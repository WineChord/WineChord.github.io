Ch7. Reasoning by Knowledge

7.1 Overview

7.1.1 Data, Information, Knowledge and Wisdom

7.1.2 Explicit and Tacit Knowledge 显性与隐性知识

  • Explicit knowledge 显性知识
    • 可表示为形式语言，包括语法陈述，数学表达式等
    • 可以快捷地转化为其他形式
    • 可以容易地用计算机语言，决策树和规则等表示
  • Tacit knowledge 隐性知识
    • 个人的经验和无形的因素，如观点等
    • 难以用形式化语言来表示
    • 神经网络提供了表示隐性知识的方法

7.1.3 Knowledge Types

Types Features
Static knowledge Unlikely to change
Dynamic Records in a database
Surface Accumulated through exp
Deep Thm/Prf/Problem Specifics
Procedural Describes how a problem is solved
Declarative Describes knowledge about a problem
Meta-knowledge Describes knowledge about knowledge
Heuristic A rule of thumb that guide the reasoning process

7.1.4 Knowledge Base and Knowledge Base System

  • 知识库--用于区分数据库
  • 知识库被用于存储复杂的结构和非结构化知识。由一套语句组成，每个语句都是由一种被称为知识表示语言来表示的，从而表示关于世界的某些断言
  • 一个知识库系统（KBS）由知识库和推理引擎组成，其中，知识库表示关于世界的事实，推理引擎则可以基于这些事实进行推理

7.1.5 Knowledge Engineering (KE)

  • 知识工程
    • 是构建，维护和使用知识库系统中所关联的所有技术，科学和社会的方方面面

7.1.6 Knowledge-based Engineering (KBE)

  • KBE是将基于知识的系统技术用于制造设计和生产领域。

7.2 Knowledge Representation

7.2.1 Overview of Knowledge Representation

  • 什么是知识表示？
    • 关注于设计计算机表示来采集关于世界的知识，可用于解决复杂的问题
    • 与过程性代码相比，使复杂的软件容易定义和维护，可用于专家系统
  • 为什么采用知识表示
    • 传统的过程性代码并非是解决复杂问题的最好形式

7.2.2 Core Issues of Knowledge Representation

  • Primitive 原语
    • 用于表示知识的基础框架，例如：语义网络，一阶逻辑等
  • Meta-representation 元表示
    • 知识表示语言用该语言本身表示
  • Incompleteness 不完备性
    • 将确定性因子与规则和结论相关联
  • Universals vs. Facts 共性与事实
    • Universals: 关于世界的一般性描述
    • Facts: 共性中的具体事例
  • Expressive adequacy 表现的充分性
    • How expressive they intend their representation to be.
  • Reasoning efficiency 推理的有效性
    • Refer to the run time efficiency of the system

7.2.3 Typical Methods of Knowledge Representation

  • Bayesian Network 贝叶斯网络
  • First Order Logic 一阶逻辑
  • Frame-based System 基于 Frame 的系统
  • Ontology 本本体
  • Production System 产生式系统
  • Script 脚本
  • Semantic Network 语义网络

7.2.4 What is Semantic Network

  • 是一种表示概念间语义关系的网络
  • 一种由结点和弧组成的有向或无向图，其中，结点表示概念，弧表示概念间的语义关系。

7.2.5 Basics of Semantic Network

  • 语义网络是基于认知的，被组织成为一个分类层次结构
  • 语义网络被采用的情形是当某种知识可以很好地化解为一组彼此相关的概念时
  • 但是难于以驾驭大型领域，并且不能很好地表现性能或元知识
  • 某些特性也不易表达：
    • 否定，析取，或者一般的非分类知识

7.3 Representation using Logic

7.3.1 Procedural vs. Declarative Approaches

  • Procedural approaches 过程性方法
    • use procedural languages, such as
      • C/C++/C#/Java
      • Lisp
      • Python
  • Declarative approaches 陈述性方法
    • use declarative languages, such as
      • Propositional logic, 命题逻辑
      • First-order logic, 一阶逻辑
      • Temporal logic 时序逻辑

7.3.2 Five Different Logics

7.3.3 Logical Symbols

7.3.4 Propositional Logic vs. First-order Logic

  • Propositional logic 命题逻辑
    • also known as propositional calculus 又称命题演算
    • use of logical connectives, deal with simple declarative propositions (if they are true or false)
  • First-order logic 一阶逻辑
    • also known as first-order predicate calculus 又称一阶谓词演算
    • additionally, use quantifiers, equality, and use predicates (often associated with sets) 额外使用了限量词，等量词以及谓词

7.3.5 Formation Rules in First Order Logic

  • The formation rules define
    • terms and formulas
  • The formation rules can be used to write a formal grammar for terms and formulas.
  • Formation rules are generally context-free, i.e.,
    • each production has a single symbol on the left side.
  • Terms:
    • Rule1: Variables
      • Any variable is a term
    • Rule2: Constants
      • Any constant is also a term
    • Rule3: Functions
      • Any expression $f(t_1,...t_n)$ of $n$ arguments is a term, where each argument $t_i$ is a term, and $f$ is a function symbol of valence $n$. In particular, symbols denoting individual constants are 0-ary function symbols, and thus terms.
  • Formulas:
    • Predicate symbols: If P is an n-ary predicate symbol and $t_1,...,t_n$ are terms, then $P(t_1,...,t_n)$ is a formula.
    • 谓词符号：若 P 是一个 n 元谓词符号并且 $t_1,...,t_n$ 是项，则 $P(t_1,...,t_n)$ 是一个公式。
    • Equality: If the equality symbol is considered part of logic, and $t_1$ and $t_2$ are terms, then $t_1=t_2$ is a formula.
    • 等量：若等量符号被认为是逻辑的一部分，并且 $t_1$ 和 $t_2$ 是项，则 $t_1=t_2$ 是一个公式
    • Negation: if $\varphi$ is a formula, then $\neg \varphi$ is a formula.
    • Binary connectives: If $\varphi$ and $\psi$ are formulas, then $(\varphi\implies\psi)$ is formula. Similar rules apply to other binary logical connectives.
    • Quantifiers: If $\varphi$ is a formula and $x$ is a variable, then $\forall x\varphi$ and $\exist x\varphi$ are formulas.

7.3.6 Prolog Language

  • Prolog 语言起源于一阶逻辑
  • 是一种通用的逻辑编程语言，已经被用于定理证明，专家系统，自然语言处理等
  • 不同于其他的编程语言，Prolog是陈述性的：程序逻辑由关系来表达，表示为事实与规则

7.4 Ontological Engineering

7.4.1 What is Ontology

  • 本体论是关于生物，生成，存在或现实的本质，以及生物及其关系的基本类别的哲学研究
  • 一个本体是一种对若干实体的类型，特性和相互关系的形式化命名和定义，它真实的，或根本地存在于一个特定范围的论域
  • 一个本体提供了一个领域的公共词汇，并且用不同层次的形式定义一些术语的含义和它们之间的关系。
  • 某些领域创建本体来组织信息，然后再将这些本体用于问题求解，包括
    • 人工智能
    • 语义Web
    • 系统工程
    • 软件工程
    • 生物医学信息学
    • 图书馆学
    • 信息架构
  • Type of Ontology
Types Concepts
Upper ontology A model of the common objects that are generally applicable across a wide range of domain ontologies.
Domain ontology Relevant to a particular topic or area of interest, e.g. information technology, or particular branches of science.
Hybrid ontology A combination of an upper ontology and a domain ontology.
  • Components of Ontology 本体的成分
Components Instances
Individuals instances or objects
Classes sets, collections, concepts, classes in programming, types of objects, or kinds of things
Attributes aspects, properties, features, characteristics, or parameters that objects(and classes) can have
Relations ways in which classes and individuals can be related to one another
Function terms complex structures formed from certain relations that can be used in place of an individual term in a statement
Restrictions formally stated descriptions of what must be true in order for some assertion to be accepted as input
Rules statements in the form of an if-then (antecedent-consequent) sentence that describe the logical inferences that can be drawn from an assertion in a partilar form
Axioms assertions (including rules) in a logical form that together comprise the overall theory that the ontology describes in its domain of application
Events the changing of attributes or relations

7.5 Bayesian Networks

7.5.1 About Uncertain Knowledge

  • 智能体的进化：问题求解，推理，规划以及学习
  • 可能需要处理不确定性，由于部分可观察性和不确定性问题
  • 由此需要：
    • 概率论
    • 效用论
    • 决策论

7.5.2 Retional Decisions

  • 概率论：用于处理置信度，degrees of belief
  • 效用论：有效性的质量，用偏好来表现和推理，每个状态都具有“有效性/效用”的度量值。
  • 决策论：理性决策的通论：
    • Decision theory = probability theory + utility theory

7.5.3 Algorithm of a Decision-theoretic Agent

7.5.4 Bayes' Rule

  • Product rule 乘积规则
    • $P(ab)=P(a|b)P(b)$ and $P(ab)=P(b|a)P(a)$
  • Bayes' rule
    • $P(b|a)=\frac{P(a|b)P(b)}{P(a)}$
  • Why works?
    • 我们常常需要根据三个项的概率估计值去计算第四个
  • 想根据一些未知原因的证据，来查明其原因
    • $P(cause|effect)=\frac{P(effect|cause)P(cause)}{P(effect)}$
  • About Bayesian Networks
    • 一种概率图模型
    • 采用一种有向无环图，表示的是一组随机变量，以及变量之间的条件相关性。
    • 规范如下：
        1. 一组结点，每个节点对应于一个随机变量
        1. 一组这对结点的有向连接
        1. 每个节点在给定双亲下的条件概率分布 $P(X_i|Parents(X_i))$
    • 同义词：
      • belief network 信念网络
      • probabilistic network 概率网络
      • causal network 因果网络
    • 一个贝叶斯网络表示一组随机变量和他们的条件依赖关系
      • 如：可以表示疾病与症状之间的概率关系
    • 两种观点：
      • 将该网络视为一种联合概率分布的表示
      • 将其视为一组条件独立语句的一种编码
    • 第一种观点有利于理解如何构建网络
    • 第二种观点有助于设计推理过程

7.5.5 Representing Full Joint Distribution

  • 采用条件分布元素的乘积：
    • $P(x_1,...,x_n)=\prod_{i=1}^nP(x_i|parents(X_i))$

7.5.6 Constructing Bayesian Networks

  • 首先，用条件概率公式对联合概率进行改写，使用如下乘积规则

    • $P(ab)=P(a|b)P(b)$
    • $P(x1,...,x_n)=P(x_n|x_{n-1},...,x_1)P(x_{n-1},...,x_1)$

  • 然后，重复该过程，将每个合取概率缩减为条件概率和较小的合取，最终得到一个大的乘积

    • $$ \begin{align*} P(x_1,...,x_n)&=P(x_n|x_{n-1},...,x_1)P(x_{n-1}|x_{n-2},...,x_1)...P(x_2|x_1)P(x_1)\ &=\prod_{i=1}^nP(x_i|x_{i-1},...,x_1) \end{align*} $$

    • 此式为任意一组随机变量的链式法则

  • 联合分布的规格等同于一般性断言，即：对于网络中的每个变量 $X_i$有

    • $$ P(X_i|X_{i-1},...,X1)=P(X_i|Parents(X_i))\ provided\ that\ Parents(X_i)\sube {X_{i-1},...,X_1} $$

  • 该等式说明：贝叶斯网络是域的一个正确表示，仅当给定其父节点，每个节点条件独立于结点序中其他前驱结点时。

  • 可用如下方法来满足该条件

      1. 结点：先确定要对域建模的变量集，${X_,...,X_n}$
      2. Links: For i = 1 to n do:
        • Choose from $X_1,...,X_{i-1}$, aminimal set of parents for $X_i$.
        • For each parent insert a link from the parent to $X_i$
        • Write down the conditional probability table (CPT)
          • $P(X_i|Parents(X_i))$
    • 结点 $X_i$ 的父辈应该包含直接影响 $X_i$ 的 $X_1,...,X_{i-1}$ 的所有节点。

7.5.7 Compactness

  • 贝叶斯网络远比全联合分布紧凑。它的紧凑性是局部结构化（也称稀疏）系统的一般特性。
  • 在局部结构化系统中，每个子成分仅直接与其他成分的有限数量打交道
  • 贝叶斯网络中，具有 $k$ 个父节点的布尔变量的 CPT (条件概率表) 有 $2^k$ 行
  • 假设有 $n$ 个布尔变量
    • Bayesian networks: 贝叶斯网络
      • 如果每个变量的父节点不超过 $k$ 个，则全部网络可以用 $n2^k$ 个数指定
    • Full joint distribution: 全联合分布
      • 包含 $2^n$ 个数

7.5.8 Node Ordering

  • 只有当选择一个好的结点排序，我们才会得到一个紧凑的贝叶斯网络

7.5.9 Conditional Independence Relations

  • Numerical semantics: 数值语义
    • 一个结点，给定其父节点后，条件独立于其他前驱结点
      • $P(x_1,...,x_n)=\prod_{i=1}^nP(x_i|parents(X_i))$
  • Topological semantics: 拓扑语义
    • 每个节点，给定其父节点后，条件独立于它的非后继结点
  • Nemerical semantics $\Leftrightarrow$ Topological semantics
  • Markov blanket 马尔可夫覆盖
    • 一个结点，给定其马尔可夫覆盖后，条件独立于网络中的所有其他结点（即：父节点，子节点，以及子节点的其他父节点）
