CS 276 索引压缩

发表于 2019-02-21 |

当当当当（sol sol sol mi），这又是一个 in-class lab，主要研究对象是索引压缩，前置知识请看第五章笔记。

参考资料：starter code , class results

两种基本的压缩方法：

  1. 传统方法：bit-level such as Huffman, Golumb, Gamma, Delta, Rice ...
  2. 对齐方法："Byte aligned" or "Word aligned" encodings, where roughly there are stretches of bits that you can use directly as numbers, and you mainly access them by masking and shifting

后一种打败了前一种方法，虽然不是压缩率最高的，但是后一种方法的解压速度更快，并且更容易实现

在索引压缩中，考虑两个点：

  1. size
  2. speed

在某种程度上，减小大小可以提高速度，但是不能太过，字节对齐所带来的速度提升要比它增加的大小要划算得多。

这个 lab 主要关注于大小

对于任何一个压缩方法，在中间都必须有一组字节，比如放在 ByteArrayOutputStream 或者 byte[] 中

bytes and bitwise operations

Java 支持十六进制和二进制表示：0xFC 0b1100

int,short,byte 分别为4，2，1个字节，可以使用强制类型转换：

有用的类：

  • BitSet

  • Simple 9 implementation

"Simple-9" 是一个 32-bit 字对齐的编码方式，它试图尽可能地将 gaps 塞到 32-bit 的字中，使用的方法是对每一个 gap 都分配相同数量的比特，并将 gap g 编码为二进制的 g-1 。

最前面的 4 个比特是用来确定一个字中要编码多少个间隔的（表见https://github.com/manning/CompressionAlgorithms），后面的 28 个比特则是来表示这些间隔的

比如，the postings list: 16384 16400 可以编码为：

0111 01111111111111 00000000001111
