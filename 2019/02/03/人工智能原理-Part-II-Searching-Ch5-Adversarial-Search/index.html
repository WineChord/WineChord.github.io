Ch5 Adversarial Search

  • To examine the problems that arise when we try to plan ahead in a world where other agents are planning against us.

5.1 Games

Search vs. Adversarial Search 搜索与对抗搜索

Search 搜索 Adversarial Search 对抗搜索
Single agent Multiple agents
Solution is (heuristic) method for finding goal Solution is strategy (strategy specifies move for every possible opponent reply)
Heuristics can find optimal solution Time limits force an approximate solution
Evaluation function: estimate of cost from start to goal through given node Evaluation function: evaluate "goodness" of game position

Adversarial Search often Known as Games 对抗搜索通常称为博弈

  • Definitions of Game theory
    • Study of strategic decision making. Specifically, study of mathematical models of conflict and cooperation between intelligent rational decision-makers.
    • An alternative term is interactive decision decision theory.
  • Applications of Game theory
    • Economics, political science, psychology, logic, computer science, and biology
    • Behavioral relations and decision science, including both humans and non-humans(e.g. computers)

Games are Good Problems for AI

  • Machines need "human-like" intelligence
  • Requiring to make decision within limited time
  • Features of games:
    • Two, or more players(agents)
    • Turn-taking vs. simultaneous moves
    • Perfect information vs. imperfect information
    • Deterministic vs. stochastic
    • Cooperative vs. competitive
    • Zero-sum vs. non-zero-sum

Zero Sum vs. Non-zero Sum

  • Zero sum games 零和博弈
  • Non-zero sum games 非零和博弈

Games are Interesting but Too Hard to Solve

Types of Games

Deterministic Stochastic
Perfect information(fully observable) Chess, Checkers, Go, Othello Backgammon, Monopoly
Deterministic Stochastic
Imperfect information(partially observable) Stratego 西洋陆军棋, Battleships 海战棋 Bridge 桥牌,Poker 扑克, Scrabble 拼字游戏

Origins of Game Playing Algorithms

  • 1912 Ernst Zermelo: Minimax algorithm
  • 1949 Claude Shannon: Chess playing with evaluation function, selective search
  • 1956 John McCarthy: Alpha-beta search
  • 1956 Arthur Samuel: Checkers program that learns its own evaluation function

Two Players Game

  • Feature
    • deterministic, perfect information, turn-taking, two players, zero-sum
  • Calling the two players:
    • MAX, MIN
  • MAX move first, and then they take turns moving, until the game is over
  • At game end
    • winner: award points
    • loser: give penalties

Formally Defined as a Search Problem

  • $S_0$ : Initial state, specifies how the game is set up at the start
  • $PLAYER(s)$ : Defines which player has the move in a state
  • $ACTIONS(s)$ : Returns the set of legal moves in a state
  • $RESULT(s, a)$ : Transition model, defines the result of a move
  • $TERMINAL-TEST(s)$: Terminal test, true when the game is over and false otherwise
  • $UTILITY(s,p)$ : Utility function, defines the value in state s for a player p

5.2 Optimal Decisions in Games

Optimal Solution 最优解

  • In normal search
    • The optimal solution would be a sequence of actions leading to a goal state (terminal state) that is a win
  • In adversarial search
    • Both of MAX and MIN could have an optimal strategy
      • In initial state, MAX must find a strategy to specify MAX's move,
      • then MAX's moves in the states resulting from every possible response by MIN, and so on.

Minimax Theorem

For every two-player, zero-sum game with finitely many strategies, there exists a value V and a mixed strategy for each player, s.t.

(a) Given player 2's strategy, the best payoff possible for player 1 is V

(b) Given player 1's strategy, the best payoff possible for player 2 is -V

  • 对于零和博弈来说，其名称 minimax 由来是因为每个玩家会使对手可能的最大收益变得最小，还会使自己的最大损失变得最小。

Optimal Solution in Adversarial Search

  • Given a game tree, the optimal strategy can be determined from the minimax value of each node, write as MINIMAX(n)
  • Assume that both players play optimally from there to the end of the game

Minimax Algorithm

Properties of Minimax Decision 最小最大决策的性质

  • 最小最大算法表现为博弈树的深度优先搜索
  • Time: $O(b^m)$
  • Space:
    • $O(bm)$ 算法同时生成所有动作
    • $O(m)$ 算法一次生成一个动作
    • b: 分支因子
    • m: 任一结点的最大深度

Optimal Decisions in Multi-player Games 多玩家博弈中最优决策

  • 将minimax思想扩展到多玩家博弈
  • 将每个节点的单一值->一个值得向量
  • 对于终端状态，从每个玩家的角度来看，这个向量给定该状态的效用。实现的最简单方法是拥有一个返回效用向量的 $UTILITY$ 函数

5.3 Alpha-Beta Pruning

Overview

  • The problem with minimax search:
    • Number of game states is exponential in depth of the tree.
    • We can't eliminate the exponent, but we can effectively cut it in half.
  • The trick to solve the problem:
    • Compute correct minimax decision without looking at every node in game tree.
    • That is, use "pruning" to eliminate large parts of the tree
  • What is alpha-beta pruning?
    • It is a search algorithm that seeks to decrease the number of nodes that are evaluated by the minimax algorithm

Why Called Alpha-Beta

  • Alpha-beta pruning gets its name from the following two parameters:
    • $\alpha$ : 沿着 MAX 路径上的任意选择点，迄今为止我们已经发现的最高值
    • $\beta$ : 沿着 MIN 路径上的任意选择点，迄今为止我们已经发现的最低值
  • Alpha-beta search respectively:
    • updates the values of $\alpha$ and $\beta$ as it goes along, and
    • prunes the remaining branches at a node as soon as the value of the current node is known to be worse than the current $\alpha$ or $\beta$ value for MAX or MIN.

Alpha-Beta Search Algorithm

5.4 Imperfect Real-time Decisions

Overview

  • minimax 算法生成整个博弈搜索空间
  • alpha-beta 剪枝算法允许我们将其剪去大部分
  • 然而，alpha-beta 仍然需要搜索抵达终端状态的所有途径，至少是搜索空间的一部分
  • 这个深度通常是不实际的，因为移动必须在合理的时间内完成

Apply a Heuristic Evaluation Function

  • 香农提出：程序应该早一些剪断搜索，并在搜索中对状态应用启发式评估函数，有效地将非终端节点转换为终端叶节点，建议使用方法：
    • 用 EVAL 来代替 UTILITY
      • EVAL: 一个启发式评价函数，用于估计位置的效用
    • 用 CUTOFF-TEST 来代替 TERMINAL-TEST
      • CUTOFF-TEST: 确定何时应用 EVAL 函数

Heuristic H-Minimax vs. Minimax

Evaluation Functions

  • 一个评价函数返回从一个给定位置该博弈的期望效用估计
    • 应与真实效用函数相同的方式对终端状态进行整理
      • 获胜状态必须评价为优于平局
      • 平局状态必须评价为优于失败
    • 计算的时间不能太长
    • 非终端状态应该与实际获胜的机会密切相关

Weighted Linear Function

$$ \begin{align*} EVAL(s)&=w_1f_1(s)+w_2f_2(s)+...+w_nf_n(s)\ &=\sum_{i=1}^nw_if_i(s) \end{align*} $$

  • $w_i$ 为权重
  • $f_i$ 为位置的特征
  • 对于国际象棋
    • $f_i$ 为棋盘上每种棋子的数量
    • $w_i$ 为棋子的数值：1 表示兵，3 表示象...

5.6 Monte-Carlo Methods

Go vs. Chess

Algorithm of AlphaGo

  • Deep neural networks
    • value network: used to evaluate board positions
    • policy network: used to select moves
  • Monte-Carlo tree search
    • Combines Monte-Carlo simulation with value networks
  • Reinforcement learning
    • used to improve its play

About Monte-Carlo Methods

  • a broad class of computational algorithms that rely on repeated random sampling to obtain numerical results
  • tend to follow a particular pattern:
    • define a domain of possible inputs
    • generate inputs randomly from a probability distribution over the domain
    • perform a deterministic computation on the inputs
    • aggregate the results

Family of Monte-Carlo Methods

  • Classical Monte-Carlo 经典
    • 样本来自于概率分布，往往是经典波兹曼分布（Boltzmann distribution）
  • Quantum Monte-Carlo 量子
    • 采用随机走查方法来计算量子力学的能量和波函数
  • Volumetric Monte-Carlo 容积式
    • 采用随机数生成的方法产生每个原子的容量，或进行其他类型几何分析
  • Kinetic Monte-Carlo 动力学
    • 仿真过程采用尺度分析来建立时间尺度，或者将随机效应引入分子动力学

Monte-Carlo Tree Search

  • 将蒙特卡洛仿真与博弈搜索树相结合
  • 和 minimax 一样，每个节点对应于一个博弈状态
  • 不同于 minimax，结点的值通过蒙特卡洛仿真来估值
  • $W(a)/N(a)=$ the value of action a
    • $W(a)=$ the total reward
    • $N(a)=$ the number of simulations

Algorithm of AlphaGo

  • 采用机器学习和树搜索技术相结合的方式，并且用人类和计算机走棋的棋局进行大量的训练
  • 两个深度神经网络
    • 价值网络来评价棋盘位置，策略网络来选择走棋
  • 树搜索
    • 蒙特卡洛树搜索
  • 强化学习
    • 用于改善其走棋

Monte-Carlo Tree Search in AlphaGo

  • (a) Selection: Each simulation traverses the tree by selecting edge with maximum action value Q + bonus u(P) that depends on a stored prior probability P for that edge.
  • (b) Expansion: The leaf node may be expanded; the new node is processed once by the policy network $p_\sigma$ and the output probabilities are stored as prior probabilities P for each action.
  • (c) Evaluation(Simulation): The leaf node is evaluated in two ways: 1) using the value network $v_\theta$; 2) by running a rollout to the end of the game with the fast rollout policy $p_\pi$ , then computing the winner with function r
  • (d) Backup(Back propagation): Action values Q are updated to track the mean value of all evaluations $r(\cdot)$ and $v_\theta(\cdot)$ in the subtree below that action.

Neural Network Training Pipeline in AlphaGo

  • A fast rollout policy $p_\pi$ and supervised learning(SL) policy network $p_\sigma$ are trained to predict human expert moves in a data set of positions
  • A reinforcement learning(RL) policy network $p_\rho$ is initialized to the SL policy network, and then improved by policy gradient learning to maximize the outcome
  • A new data set is generated by playing games of self-play with the RL policy network
  • A value network $v_\theta$ is trained by regression to predict the expected outcome

Neural Network Architecture in AlphaGo

  • The policy network 策略网络

    • input: the board position
    • passes s through convolutional layers with parameters $\sigma$ or $\rho$
    • outputs: a prbability distribution over legal moves a.

  • The value network 价值网络

    • input: the board position s'
    • similarly uses many convolutional layers with parameters $\theta$
    • output: a scalar value $v_\theta(s')$ that predicts the expected outcome.
