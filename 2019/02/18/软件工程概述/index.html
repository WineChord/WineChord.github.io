软件工程概述

发表于 2019-02-18 |
  • 软件=程序+数据+文档
  • 具有复杂性，一致性，可变性和不可变性
  • 产生与发展：
    • 史前时代：1956-1967
    • 瀑布过程模型：1968-1982
    • 质量标准体系：1983-1995
    • 敏捷开发：20世纪90年代至今
  • 基本策略：软件复用，分而治之，逐步演进，优化折中
  • Python 编程规范：程序模板

  • 注释
    • 当前实现的原因，以"#"开头
    • 文档字符串，说明如何使用，甚至包括示例和单元测试

  • 注重注释的质量而不是数量，解释为什么而不是怎么样

  • 语句：

    • 使用四个空格来进行缩进

    • 类名为驼峰风格且首字母大写（如：BookShelf）

    • 变量名应为下划线风格（如：book_shelf）

    • 使用文件时注意显式调用 close() 或使用 with：

    • 每行写一个语句

  • import：先 import Python 内置模块，再 import 第三方模块，最后 import 自己开发的其他模块

  • 不要使用 from module import *，除非是 import 常量定义模块或其他能够确保不会出现命名空间冲突的模块
