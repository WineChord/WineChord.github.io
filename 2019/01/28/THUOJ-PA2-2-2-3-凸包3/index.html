THUOJ PA2 2-2-3 凸包3

发表于 2019-01-28 | 分类于 THUOJ |

问题描述

  • 平面上给 $n$ 个点，求凸包
  • $n$ 可能很大
  • 输入数据已经按照点的横坐标大小进行了排序

输入输出

  • 和凸包2一样，输入文件仍采用二进制格式存储
  • 输出时选择凸包上纵坐标最小（纵坐标相等则选横坐标最小的）的点作为第一个输出的点，按逆时针顺序依次输出凸包上的所有点。
  • 若有重合的点，只需输出一次，不需输出凸包上非顶点的点。

输入输出

限制

  • $5\le n \le 10,000,000$
  • $-2^{25} \le x,y \le 2^{25}$

提示

  • 数据特点：点的数量可能很大，输入数据已按点的横坐标从小到大排序
  • 适用算法：由于点的数量很大，使用复杂度为VH的JarvisMarch算法和复杂度为VlogV的快速凸包算法均会超时。可以使用GrahamScan算法，该算法在数据有序的情况下复杂度为V，其中V为点的总数。

思路

  • 使用 $GrahamScan$ 算法。

GrahamScan 算法

  • 参考wiki
  • 首先找到有最小 $y$ 坐标的点 $P$（$y$ 坐标相同则找具有最小 $x$ 坐标的点）$O(n)$
  • 然后将所有的点按照与 $P$ 点形成的极角大小进行排序 $O(nlogn)$
    • 算极角时不一定非要算出来角度，只要能比较出大小即可。可以使用任何在 $[0,\pi]$ 之间单调的函数来进行衡量，比如可以用 $cos(x)$ 函数
  • 当三个点形成“右转”关系时，说明中间那个点不是凸包点，舍弃该点，再取下一个点进行判断。
  • 三点共线时，根据需要进行取舍。
  • 判断三点关系，实际上就是判断两个向量的顺逆时针关系，举例如下：
    • 考虑三个点 $P_1=(x_1,y_1)$ $P_2=(x_2,y_2)$ $P_3=(x_3,y_3)$
    • 考虑两个向量 $\overrightarrow{P_1P_2}$ $\overrightarrow{P_1P_3}$
    • 两个向量的叉积为 $(x_2-x_1)(y_3-y_1)-(y_2-y_1)(x_3-x_1)$
    • 如叉积等于0，则三点共线
    • 若叉积大于0，则向量形成逆时针关系，为“左转”
    • 若叉积小于0，则向量形成顺时针关系，为“右转”
