THUOJ PA1 1-2 Company

发表于 2019-01-23 | 分类于 THUOJ |

问题描述

  • 公司有n个员工，编号为1~n。
  • 员工上班时登陆管理系统登记一个code，离开时进行注销。
  • 管理人员随时想知道多少员工上班以及任一个员工登记的code。

输入输出

  • 第一行是两个整数n,m。接下来是m行，每行是以下内容之一：
    • I a c // Log In：员工 a 登录，code 为 c。若 a 已登录，则将 code 更新为 c
    • O a // Log Out：编号为 a 的员工注销。若 a 未登录，则注销无效
    • C // Close：到下班时间，所有员工注销
    • N // Number：询问有多少员工正在上班
    • Q a // Query：询问员工 a 的 code（若未登录或已注销，视为 -1）

输入输出举例：

  • 限制
    • n <= 10,000,000
    • m <= 1,000,000 1 <= a <= n code 为 [0, 2^31) 的整数

提示

一级提示

测试数据中，每种操作都可能频繁出现，或很少出现。因此，你的算法需要应对所有可能的情况。

答案会超出 int 范围，要注意各种数据类型可表示的范围，尤其是溢出问题。可选用 long long int 或 int64_t 等类型。注意各种数据类型的正确输入、输出方式。

  • Bitmap / 常数时间初始化的数组——讲义 02XA
  • Bitmap: 习题解析 [2-34]

二级提示

  • 避免频繁初始化整个数组。
  • 可以借助Bitmap记录数组的各元素是否已被初始化。将一个（或所有）元素置为未初始化状态，实质上只需要打破“校验环”两个验证条件中的任何一个，考虑如何打破校验条件的复杂度最低。
  • 另一种解题思路是，数组初始化 1 次后，记录改变了的元素，今后不再初始化整个数组，只初始化改变了的元素。

用 scanf("%c", ...) 或 getchar() 读入一个字符时，可能会读到空格、制表符、回车符或换行符，尽量避免使用这个方法。使用 scanf("%s", ...) 会自动过滤空白字符。

Windows 和 Linux 上换行符有 "\r\n" 跟 "\n" 的差别，你的程序最好有一定鲁棒性，能处理两种情况。

思路

  • 先实现低效版，再思考改进。

低效版

  • 先写一个低效版。其中通过while (isspace(op)) scanf("%c", &op);来滤去空格以及换行制表等符号。
  • 算法低效之处主要在于close()函数，当下班时只能对所有的元素进行赋值-1，而不能仅仅只针对已上班的员工。其他的主要函数时间复杂度都是O(1).
  • 总的时间复杂度为O(n)，空间复杂都为O(n)
  • 除了9,10,19测试点超时，其余测试点均可以通过。
  • 将此算法生成的执行文件作为对拍的标准程序。

思考改进

  • 考虑加上习题解析[2-34]中bitmap的思路
  • 采用同时支持set() clear()这两样操作的类，参考代码x2.9。
  • 可以在O(1)时间隐式初始化（但是还有至少要有O(n)的分配内存的时间的）
  • 查上班人数时还是沿用低效版的思路，用一个变量表示上班人数，log_in时加一，log_out时减一，都下班时清零。
  • 仿照bitmap设置两个向量，F[]来记录[k]被标记的次序，向量T[]来记录依次被标记的值。
  • clear()时的改进：只需将top置零，时间复杂度O(1)
  • 同时，code[]也不用全部初始化，只需每次修改code[F[k]]即可
  • 如此，所有的操作时间复杂度都是O(1)
