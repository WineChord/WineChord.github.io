CST2018 3-4 kth

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 三个数组 a[1...n], b[1...n], c[1...n]，在所有数对$(x,y,z)$ $(1\le x,y,z\le n)$ 中，找到 $a[x]+b[y]+c[z]$ 的值第 $k$ 小的那对，若有多对，找出任何一对即可
  • 不能直接访问三个数组中的元素，只能给出两种方案，两个数对，通过函数接口，来比较值得大小

接口说明

  • kth.h
  • kth_lib.cpp

  • 需要实现 get_kth，找到第 $k$ 小的数对后，需要将该数对的三个值分别存在 *x,*y,*z 中

  • 当 $a[x]+b[y]+c[z]\lt a[u]+b[v]+c[w]$ 时，返回 $1$ ，否则返回 $0$
  • compare 的实现为 $O(1)$ ，可认为每秒钟至少能调用 1e7 次该函数

测试说明

  • 首先读入 $n\ k$
  • 然后读入并生成 a,b,c 数组中的数
  • 接着调用 get_kth 函数
  • 在实现的 get_kth 函数中，可以调用 compare 函数来对两个数进行比较，若传入参数不在 $1\sim n$ 的范围内，判为 WA
  • get_kth 返回后，输出 $a[x]+b[y]+c[z]$ 的值
  • 本地测试：g++ kth_lib.cpp your_code.cpp -O2 -o kth.exe
  • 样例程序：kth_sample.cpp

输入输出

  • 输入：
    • 第 $1$ 行：$n\ k$
    • $n$ 个整数，第 $i$ 个表示 $a[i]$
    • $n$ 个整数，第 $i$ 个表示 $b[i]$
    • $n$ 个整数，第 $i$ 个表示 $c[i]$
    • 每个整数 $\in [1,10^8]$
  • 输出：
    • 正确时输出方案的值，错误时输出错误信息

限制

  • 4 sec
  • 256 MB

提示

  • [n 特别大，枚举所有方案好像不太可能]

  • [如果只有一个数组，要求第 k 小，可以怎么求？]

  • [如果有两个数组，该怎么求？]

  • [如果有三个数组呢？　　　　]

  • [堆　　　　　　　　　　　　]

$$ P(l,r)=\max_{l\le u,v\le r}S(u,v) $$

为多少？

  • 输出：

    • 每行输出对应最大子区间和的大小

数据规模及限制

  • $1\le n\le 500,000$
  • $1\le m\le 500,000$
  • $-200\le value, ai\le 200$
  • 2 sec
  • 256 MB

提示

  • 线段树, 考虑线段树中的一条线段节点r，如果目标线段a被他包含，那么目标线段a对于线段树中的r节点来讲有三种可能，在左子树(左半线段)，右子树(右半线段),或者经过线段中点(左半线段的右侧+右半线段的左侧)

CST2018 2-6 Temperature

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 提供气温数据查询服务
  • 根据用户指定矩形区域内所有观察站的观测值计算出平均气温
  • 请改进数据结构和算法，提高查询效率
  • 观察站的坐标呈矩阵排列，气象台可以随时修改任意一个观测站的气温
  • 用户可以指定某矩形区域来获得该区域内所有观察站的平均气温

接口说明

  • 头文件 temperature.h
  • 库文件 temperature_lib.cpp
  • 请实现初始化函数和查询函数：

  • 此为初始化函数的声明，可以在此进行初始化
  • 其中 $n,m$ 表示观察站构成矩形的行数和列数，$temp$ 为数组的数组，其中 $temp[i][j]$ 表示第 $i$ 行第 $j$ 列观察站的初始气温。

  • 此为查询函数的声明。初始化完成后，交互库会调用该函数多次，每一次需要进行相应查询并返回结果
  • 查询的四边分别与 $x$ 轴或 $y$ 轴平行，其中 $(x1,y1)$ 和 $(x2,y2)$ 为一对角线，在矩形边界上的观察站也要记录。
  • 返回平均气温的数值，进行截断取整

  • 此为修改函数的声明，在初始化完成后，交互库会调用该函数若干次，需要将坐标为 $(x,y)$ 的观察站处的温度修改为 $temp$。

测试说明

  • 首先从标准输入中读取数据
  • 然后调用 $init$ 函数
  • 接着对于每个不同的操作相应地调用 $query$ 和 $change$ 函数，并将 $query$ 函数的结果写入标准输出
  • 本地测试：g++ temperature_lib.cpp your_code.cpp -O2 -o temperature.exe
  • 样例程序 temperature_sample.cpp 实现了“每次查询返回一个随机数”的算法

输入输出

  • 输入：

    • 第 $1$ 行：$n\ m\ k$ 观察站函数和列数，操作总数
    • 以下一行为 $n*m$ 个整数，分别表示坐标为 (1,1),(1,2),...,(1,m),(2,1),(2,2),...,(n,1),(n,2),...,(n,m) 的观察站的初始温度。
    • 以下 $k$ 行：每行若干整数，表示一个操作

  • 输出：

    • 若干行：每行对应一次求平均气温的查询结果（截断取整后）

数据范围及限制

  • 气温为整数 $\in [-1000,1000)$

  • $1\le n\le 1200$

  • $1\le m\le 1200$

  • $1\le k\le 2,000,000$

  • time：3 sec

  • space：256 MB

提示

  • 仿照 range tree 的思路， 第二个维度的树的每个节点保存区间和，以便快速计算区间和

  • 因为这题坐标是连续的，因此不需要使用 range tree 的树结构，只需要用数组存储线段树即可

  • 本题只有单点修改，如果你的常数优化不够好，可以自学数组状的树（又称“树状数组”）完成优化
