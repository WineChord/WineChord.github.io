CST2018 2-5 ChromPoly

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 考察无向图 $G=(V,E)$，$G$ 的一个 $k-$染色（coloring）是从 $V$ 到 $[1,k]$ 的一个映射：

    $\pi:V\rightarrow {1,2,...,k}$

    且满足

    $\forall v,u\in V,\pi(v)\not= \pi(u) if (v,u)\in E$

  • 即：将所有顶点归入不超过 $k$ 个等价类，且保证相邻顶点不属同类

  • 若将图 $G$ 和 $k$ 视作变量，则可用函数 $CP(G,k)$ 表示图 $G$ 的 $k-$染色方案数。而对于任一图 $G$，该函数都是 $k$ 的多项式形式，称作色多项式（chromatic polynomial）

  • 例子：

  • 对于包含 $n$ 个顶点，不含任何边的图 $Sn$，有

    • $CP(Sn,k)=k^n$

  • 对于完全图 $Kn$，有

    • $CP(Kn,k)=0,if\ k\lt n$
    • $CP(Kn,k)=\frac{k!}{(k-n)!},if\ k\ge n$

  • 对于二部图（bipartition）$B(2,3)$，有（在这个语境里其实是完全二分图）

    • $CP(B(2,3),1)=0$
    • $CP(B(2,3),2)=2$
    • $CP(B(2,3),3)=6(2*3)+6\times 8\div 2=30$
    • $CP(B(2,3),k)=k^5-6k^4+15k^3-17k^2+7k$

  • 对于 5 阶环 $C5$，有

    • $CP(C5,1)=P(2,C5)=0$
    • $CP(C5,3)=30$
    • $CP(C5,k)=k^5-5k^4+10k^3+4*k$

  • 请编写程序，求对于给定的图 $G$，计算并输出对应的色多项式 $CP(G,k)$

输入输出

  • 输入：
    • 第 $1$ 行：$n\ m$ 顶点数和边数
    • 以下每行：$(u,v)$表示图中的一条边，且保证满足 $0\le u,v\lt n$
  • 输出：
    • $n+1$ 行：每一行表示该图色多项式各项的系数（依 $k$ 的次数由低向高排列）

数据范围

  • $1\le n\lt 30$
  • $1\le m\lt 70$
  • 保证图中无自环
  • 各项系数的值保证在带符号64位整数范围内
  • time：7 sec
  • space：1024 MB

说明

  • $CP(G,k)=CP(G\backslash {e},k)-CP(G/{e},k)$
  • 其中 $\backslash$ 为减除（subtraction）操作，即为从图中将某一边抹去。
  • $/$ 为收缩（contraction）操作，即为收缩边 $e$ ，直到其顶点 $v$ 和 $u$ 粘连为同一个顶点 $w$；而与之关联的边除 $e$ 外依然保留。
  • 更多：https://dsa.cs.tsinghua.edu.cn/~deng/ds/00240074@2004spring/handout/assignment/chromaticpolynomial/

CST2018 2-4 Sort

发表于 2019-02-09 | 分类于 THUOJ |

问题描述

  • 每次比较三个数，返回最小数和最大数
  • 为测试这个比较器，需要实现一个排序算法，使用该比较器，将 $n$ 互不相同的整数从小到大排序
  • 提供了一个交互库，其中实现了一个比较器函数，需要编写一个排序函数，调用比较器函数将序列排好序
  • 需要在比较器使用次数限制之内，将这些整数正确地排序

接口说明

  • 提供的交互库包括头文件sort.h和库文件sort_lib.cpp，头文件包含排序函数和比较器函数的生命，库文件中包含比较器函数和主函数的实现，需在程序中包含头文件 sort.h并实现排序函数

  • 为排序函数的声明，需要实现。$n​$ 为待排序的个数，limit 表示比较器函数的使用次数限制，a为一个长度为n整数数组，初始值为 $0\sim n-1​$。需要将排序的结果存入 a 数组中

  • 无法得到待排序的整数的值，而只能得到每个整数的编号。因此 a 数组中的值，以及 a,b,c 都应该是 $0\sim n-1$ 之间的数，表示某个整数的编号。

测试说明

  • 交互库具体实现如下：
    • 首先从标准输入中读取两个整数 $n,limit$，分别表示待排序的整数个数和比较器函数的使用次数限制。
    • 然后以某种方式读入并生成待排序的 $n$ 个整数
    • 接着调用 sort 函数
    • 在你实现的 sort 函数中，可以调用 compare 函数来进行比较，如果传入的 a,b,c 不在 0~n-1 之间，或者调用次数超过 limit，会 WA
    • sort 函数返回后，会检查 a 数组的内容是否正确排序
    • 本地测试：g++ sort_lib.cpp your_code.cpp -O2 -o sort.exe来编译，然后运行 sort.exe 来测试
    • 还提供了一个样例程序 sort_example.cpp实现了 Bogo 排序算法

输入输出

  • 输入：
    • 第 $1$ 行：$n\ limit$
    • 第 $2$ 行：$n$ 个整数，要求是 $0\sim n-1$ 的排列，表示待排序的序列
  • 输出：
    • 正确时："Accepted"
    • 错误时：一些错误信息

提示

  • [有哪些常用的排序算法？这些算法的期望、最好、最坏比较次数分别是多少？]

  • [给 n 个数排序的比较次数的理论下界是多少？如果一次能比较三个数，又是多少？]

  • [哪些排序算法可以改进成一次比较三个数的？改进效果分别如何？]

  • [提示：对已有的排序算法作更多的改进，以进一步减少比较次数]

CST2018 1-7 FIFO

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 简化版的FIFO调度算法
    1. 访问某个地址的值时，先在缓冲器中寻找该地址及其对应值
    2. 若在缓冲器中找到该地址，则将其对应值返回，操作结束
    3. 若未找到，则从内存中读取该地址对应值，并试图将该地址及其对应值存到缓冲器中
    4. 若缓冲器未满，则存，操作结束
    5. 若满，删除最先进入缓冲器的元素，添加当前访问地址及其值
  • 刚开始缓冲器为空，现有一系列访存请求，问有几次需要去内存中访存

输入输出

  • 输入：
    • 第 $1$ 行：$n\ m$ 缓冲器的大小和访存次数
    • 以下一行：$m$ 个整数，表示每次访问的地址，每个地址是1-100的整数
  • 输出：
    • 一个整数，表示答案

数据范围及限制

  • $1\le n\le 100$

  • $1\le m\le 1000$

  • $1\le 内存地址\le 100$

  • 时间：1 sec

  • 空间：256 MB

提示

题目描述

  • 一个完美的文本编辑器，拥有两个光标，可以在两个地方插入和删除文本。
  • 功能列表如下：
功能名称 命令格式 说明
< (move left) < w w为一个字符，不是'L'就是'R'，代表左光标还是右光标。将选定光标向左移动，如果是最左端则不移动。执行成功输出"T"，否则输出"F"（已在最左侧）
> (move right) > w 类似上
I (insert) I w c c是一个可见字符（$33\le ascii码\le 126$），表示在该光标左插入该字符，该命令始终输出"F"
D (delete) D w 表示删除该光标右的一个字符，成功输出"T"，否则输出"F"（光标右侧没有字符）
R (reverse) R 翻转左光标和右光标之间的字符，只有左光标在右光标的左侧时才能执行（两光标重合时也不能执行）成功时输出 "T"，否则 "F"
S (show) S 显示当前处理的文本，只输出文本，不输出"T"或"F"
  • 开始时文本编辑器中有一定内容，左光标在第一个字符左，右光标在最后一个字符右。
  • 注：插入和删除操作中，没有被操作的光标与文本的相对左右位置保持不变。特别地，若两个光标重叠，操作后也仍然重叠。

输入输出

  • 输入：
    • 第 $1$ 行：初始文本编辑器的内容
    • 第 $2$ 行：$N$ 操作次数
    • 以下 $N$ 行：每行一个命令
  • 输出：
    • 对于每个命令执行并输出

数据范围及限制

  • 初始字符，插入字符是可见字符（$33\le ascii 字符\le 126$）
  • $1\le 初始文本长度\le 3,200,000$
  • $1\le N\le 4,000,000​$
  • 输出文件大小$\le 20MB$

  • 列表。

  • 使用数组集中存放链表节点可以提高效率。
