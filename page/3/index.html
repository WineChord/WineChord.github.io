思路

  • 根据提示，使用快速凸包算法
  • 本题输入方式和之前的不一样，先实现一般版本，再根据题目要求进行改进。

快速凸包

  • 采用分治法，平均时间复杂度为$O(nlog(n))$，最坏时间复杂度为$O(n^2)$。
  • 主要分为以下几步（参考wiki）：
    • 找到具有最小和最大的x坐标的点 A 和 B
    • 线 AB 将点集划分为两部分，用来之后的分而治之
    • 在每一侧都找到距离 AB 最远的点，这些点和直线在两边分别形成了一个三角形
    • 在三角形内部的点肯定不是凸包点
    • 在三角形除了AB的两边重复以上两步
    • 重复直到没有剩余的点，被选中的点即形成了凸包

计算点到直线的距离

  • Python 实现中的findR(Sk, P, Q)要求找到Sk中距离PQ最远的点，需要计算点到直线的距离
  • 参考文献
  • 取点 $O$，
    • $\overrightarrow{a} = \overrightarrow{PO}$
    • $\overrightarrow{b} = \overrightarrow{PQ}$

$$
\overrightarrow{c} = \frac{\overrightarrow{a} \cdot \overrightarrow{b}}{|\overrightarrow{b}|} \cdot \frac{\overrightarrow{b}}{|\overrightarrow{b}|} = \frac{\overrightarrow{a} \cdot \overrightarrow{b}}{|\overrightarrow{b}|^2} \cdot \overrightarrow{b}
$$

$$
|\overrightarrow{e}| = |\overrightarrow{a} - \overrightarrow{c}|
$$

THUOJ PA1 1-4 Alarm

发表于 2019-01-24 | 分类于 THUOJ |

题目描述

  • 如图，图中的线为红外线，所有红外线不相交。
  • n个发射器和n个接收器将平面分为n+1个区域，从左到右记为0,1…n
  • 判断给定的点在哪个区域上。若正好在红外线上，则认为处于右边的区域。

输入输出

  • 输入
    • 第一行：n m
    • 以下n行：每行a b，表示发射器在(a,0)，接收器在(0,b)，每行a,b都比前一行大
    • 以下m行：每行x y，表示求(x,y)所在的区域
  • 输出
    • m行，每行一个数，代表该点所在区域

输入输出样例

限制

  • 1<=n<=200,000
  • 1<=m<=200,000
  • 1<={a,b,x,y}<2^31

提示

  • 可以用向量积判断点与直线的位置关系 http://www.cnblogs.com/bfshm/articles/3620500.html
  • 普通的遍历搜索是不能满足时间要求的，考虑使用二分查找等方法。
  • 一些中间结果（例如向量积做乘法后的中间结果）可能会超出 int 范围，参考编程作业说明“附：一些常见编程错误”节。

    编程作业说明 附：一些常见编程错误 :计算溢出。错误示例：
    int a = 1000000, b = 1000000;
    long long c = a * b;
    程序会先在 int 范围内计算 a b，最后把溢出后的结果赋值给 c。正确写法是 `c = (long long)a (long long)b`。

    思路

  • 根据两个点建立起直线的方程，然后根据向量积判断点和直线的位置关系，找到该点在哪两条线之间，从而判断在哪个区域
  • 关键在于怎么快速地判断在哪两条线之间，根据提示，使用二分查找（二分查找的灵活应用还是挺多的，比如2017THU机试）

点和直线的位置关系的判断方法

  • 利用矢量叉积：
    • $\boldsymbol{P} = (x_1, y_1)$,
    • $\boldsymbol{Q} = (x_2, y_2)$,
    • $\boldsymbol{P} \times \boldsymbol{Q} = x_1y_2 - x_2y_1$
    • 若$\boldsymbol{P} \times \boldsymbol{Q} > 0$，则$\boldsymbol{P}$在$\boldsymbol{Q}$的顺时针方向。
    • 若$\boldsymbol{P} \times \boldsymbol{Q} < 0$，则$\boldsymbol{P}$在$\boldsymbol{Q}$的逆时针方向。
    • 若$\boldsymbol{P} \times \boldsymbol{Q} = 0$，则$\boldsymbol{P}$与$\boldsymbol{Q}$共线。
  • 由此，设点为$O$，$x$轴上的点为$A$，$y$轴上的点为$B$，令：
    • $\boldsymbol{P} = \overrightarrow{OA}$
    • $\boldsymbol{Q} = \overrightarrow{OB}$
  • 从而有：
    • 令$s = \boldsymbol{P} \times \boldsymbol{Q}$
    • $s>0 \Rightarrow$ 点在线的左侧
    • $s<0 \Rightarrow$ 点在线的右侧

THUOJ-PA1-1-3-2-Super Interview

发表于 2019-01-24 | 分类于 THUOJ |

问题描述

  • 选出n个人进行面试，每个应聘者被分配了一个整数ID。要注意人数非常多（数据量很大）
  • 根据以下规则决定面试顺序
  • 来一张圆桌，第一个到的人随意坐
  • 之后每一个到的人从前一个人出发逆时针走过m个人（前一个人算第一个），然后紧邻第m个人的右侧就座。所有人入座后，从最后到达者出发，绕圆桌顺时针面试。
  • 请确定面试顺序。

输入输出

  • 输入：
    • 第一行：n m
    • 第二行：n个整数，表示先后到达的人的ID
  • 输出：
    • 共一行：以空格分隔的n个整数，分别是按顺序面试的ID

测试用例

限制

  • 1<=n<=10^6
  • 1<=n*m<=10^7
  • 输入的ID保证在int类型的范围内

提示

一级提示

  • 循环链表

二级提示

  • 因为找座位按逆时针方向，面试按顺时针发现，所以建议使用双向循环链表。
  • 往链表中插入元素时，要小心地修改前驱指针和后继指针。
  • 一次性new n个节点，比分n次每次new一个节点快一些。

思路

  • 用循环链表进行模拟
  • 很像约瑟夫问题

实现

  • 最初版本，可通过所有测试用例，worst case 是19，time:712，memory:51524
  • 关键在于实现一个双向链表，难度不大。
  • 排行榜最好情况是 time:128 memory:24152
  • 加上两行setvbuf(stdin, new char[MAXSIZE], _IOFBF, MAXSIZE);
    setvbuf(stdout, new char[MAXSIZE], _IOFBF, MAXSIZE);（MAXSIZE=4000000 + 5）之后，最坏测例19的时间是472，空间是59332。加快了一倍。

THUOJ PA1 1-2 Company

发表于 2019-01-23 | 分类于 THUOJ |

问题描述

  • 公司有n个员工，编号为1~n。
  • 员工上班时登陆管理系统登记一个code，离开时进行注销。
  • 管理人员随时想知道多少员工上班以及任一个员工登记的code。

输入输出

  • 第一行是两个整数n,m。接下来是m行，每行是以下内容之一：
    • I a c // Log In：员工 a 登录，code 为 c。若 a 已登录，则将 code 更新为 c
    • O a // Log Out：编号为 a 的员工注销。若 a 未登录，则注销无效
    • C // Close：到下班时间，所有员工注销
    • N // Number：询问有多少员工正在上班
    • Q a // Query：询问员工 a 的 code（若未登录或已注销，视为 -1）

输入输出举例：

  • 限制
    • n <= 10,000,000
    • m <= 1,000,000
      1 <= a <= n
      code 为 [0, 2^31) 的整数

提示

一级提示

测试数据中，每种操作都可能频繁出现，或很少出现。因此，你的算法需要应对所有可能的情况。

答案会超出 int 范围，要注意各种数据类型可表示的范围，尤其是溢出问题。可选用 long long int 或 int64_t 等类型。注意各种数据类型的正确输入、输出方式。

  • Bitmap / 常数时间初始化的数组——讲义 02XA
  • Bitmap: 习题解析 [2-34]

二级提示

  • 避免频繁初始化整个数组。
  • 可以借助Bitmap记录数组的各元素是否已被初始化。将一个（或所有）元素置为未初始化状态，实质上只需要打破“校验环”两个验证条件中的任何一个，考虑如何打破校验条件的复杂度最低。
  • 另一种解题思路是，数组初始化 1 次后，记录改变了的元素，今后不再初始化整个数组，只初始化改变了的元素。

用 scanf(“%c”, …) 或 getchar() 读入一个字符时，可能会读到空格、制表符、回车符或换行符，尽量避免使用这个方法。使用 scanf(“%s”, …) 会自动过滤空白字符。

Windows 和 Linux 上换行符有 “\r\n” 跟 “\n” 的差别，你的程序最好有一定鲁棒性，能处理两种情况。

思路

  • 先实现低效版，再思考改进。

低效版

  • 先写一个低效版。其中通过while (isspace(op)) scanf("%c", &op);来滤去空格以及换行制表等符号。
  • 算法低效之处主要在于close()函数，当下班时只能对所有的元素进行赋值-1，而不能仅仅只针对已上班的员工。其他的主要函数时间复杂度都是O(1).
  • 总的时间复杂度为O(n)，空间复杂都为O(n)
  • 除了9,10,19测试点超时，其余测试点均可以通过。
  • 将此算法生成的执行文件作为对拍的标准程序。

思考改进

  • 考虑加上习题解析[2-34]中bitmap的思路
  • 采用同时支持set() clear()这两样操作的类，参考代码x2.9。
  • 可以在O(1)时间隐式初始化（但是还有至少要有O(n)的分配内存的时间的）
  • 查上班人数时还是沿用低效版的思路，用一个变量表示上班人数，log_in时加一，log_out时减一，都下班时清零。
  • 仿照bitmap设置两个向量，F[]来记录[k]被标记的次序，向量T[]来记录依次被标记的值。
  • clear()时的改进：只需将top置零，时间复杂度O(1)
  • 同时，code[]也不用全部初始化，只需每次修改code[F[k]]即可
  • 如此，所有的操作时间复杂度都是O(1)

问题描述

  • 教练ABC从编号为1-n的队员中选队员，根据他们各自的喜好来进行排序
  • a,b,c三个数组表示三个教练对队员的喜好程度
  • 每个数组为1-n的一个排列
  • 喜好程度从高到低：a[0],a[1]…a[n-1]
  • 分配规则：
    • 从未分配队员中找A最喜欢的队员分之
    • 然后再找B,C的

  • 问队员k分给了哪个教练？

  • 限制：

    • 1 <= n <= 500,000
    • 1 <= k <= n
