题目描述

  • 电路板可以插两个元件，每个元件64个引脚，每个引脚输出0或1，64位输出会按某种固定的顺序拼成表示信号强度的数。电路板由64个异或门组成，即有64个输出引脚，每一位是两个元件对应位的异或。

限制

  • $2\le n\le 500,000$
  • $0\le k\le 500,000$

提示

CST2018 1-7 FIFO

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 简化版的FIFO调度算法
    1. 访问某个地址的值时，先在缓冲器中寻找该地址及其对应值
    2. 若在缓冲器中找到该地址，则将其对应值返回，操作结束
    3. 若未找到，则从内存中读取该地址对应值，并试图将该地址及其对应值存到缓冲器中
    4. 若缓冲器未满，则存，操作结束
    5. 若满，删除最先进入缓冲器的元素，添加当前访问地址及其值
  • 刚开始缓冲器为空，现有一系列访存请求，问有几次需要去内存中访存

输入输出

  • 输入：
    • 第 $1$ 行：$n\ m$ 缓冲器的大小和访存次数
    • 以下一行：$m$ 个整数，表示每次访问的地址，每个地址是1-100的整数
  • 输出：
    • 一个整数，表示答案

数据范围及限制

  • $1\le n\le 100$

  • $1\le m\le 1000$

  • $1\le 内存地址\le 100$

  • 时间：1 sec

  • 空间：256 MB

提示

  • 队列

CST2018 1-6 9的完美文本编辑器

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 一个完美的文本编辑器，拥有两个光标，可以在两个地方插入和删除文本。
  • 功能列表如下：
功能名称 命令格式 说明
< (move left) < w w为一个字符，不是'L'就是'R'，代表左光标还是右光标。将选定光标向左移动，如果是最左端则不移动。执行成功输出"T"，否则输出"F"（已在最左侧）
> (move right) > w 类似上
I (insert) I w c c是一个可见字符（$33\le ascii码\le 126$），表示在该光标左插入该字符，该命令始终输出"F"
D (delete) D w 表示删除该光标右的一个字符，成功输出"T"，否则输出"F"（光标右侧没有字符）
R (reverse) R 翻转左光标和右光标之间的字符，只有左光标在右光标的左侧时才能执行（两光标重合时也不能执行）成功时输出 "T"，否则 "F"
S (show) S 显示当前处理的文本，只输出文本，不输出"T"或"F"
  • 开始时文本编辑器中有一定内容，左光标在第一个字符左，右光标在最后一个字符右。
  • 注：插入和删除操作中，没有被操作的光标与文本的相对左右位置保持不变。特别地，若两个光标重叠，操作后也仍然重叠。

输入输出

  • 输入：
    • 第 $1$ 行：初始文本编辑器的内容
    • 第 $2$ 行：$N$ 操作次数
    • 以下 $N$ 行：每行一个命令
  • 输出：
    • 对于每个命令执行并输出

数据范围及限制

  • 初始字符，插入字符是可见字符（$33\le ascii 字符\le 126$）
  • $1\le 初始文本长度\le 3,200,000$
  • $1\le N\le 4,000,000​$
  • 输出文件大小$\le 20MB$

提示

  • 列表。

  • 使用数组集中存放链表节点可以提高效率。

CST2018 1-5 Zuma

发表于 2019-02-09 | 分类于 THUOJ |

题目描述

  • 一条轨道上初始有 $n$ 个彩色珠子，其中任意三个相邻的珠子不会完全同色。
  • 可以发射珠子到轨道上，一旦有三个或更多同色的珠子相邻，就会立即消失。可发生连锁反应
  • 请计算一系列操作之后的珠子序列

输入输出

  • 输入：
    • 第 $1$ 行：由 'A'-'Z' 组成的字符串，表示初始珠子序列
    • 第 $2$ 行：$m$ 表示共有 $m$ 次操作
    • 以下 $m$ 行：每行代表一次操作，每次操作为 $k\ \Sigma$ ，其中 $k\in[0,b]$ 表示新珠子嵌入之后在轨道上的位序，$\Sigma$ 表示新珠子的颜色
  • 输出：
    • 共 $1$ 行：给出所有操作之后轨道上的珠子序列
    • 若没有珠子，则输出空行（一个换行符）

数据范围及限制

  • $0\le n\le 500,000$

  • $0\le m\le 500,000$

  • 输入保证位置合法

  • 时间：2 sec

  • 空间：256 MB

提示

  • 本题字母、数字混合输入，且初始序列可能为空，请使用足够鲁棒的读入方式。

  • [ 我们将介绍一种简单高效的数据结构，分块数组。他是一个结合了链表和列表的高效的数据结构。 ]

  • [ 列表可以 $O(1)$ 完成寻址，但是需要 $O(N)$ 完成删除和插入，链表可以 $O(1)$ 完成删除和插入，但是需要 $O(N)$ 完成寻址。我们将他们结合起来选择 $\sqrt{n}$ 为一个中间值。 ]

  • [ 将 $N​$ 长度的列表以 $O(\sqrt{n})​$ 长度断成 $O(\sqrt{n})​$ 块，每块用链表连起来，这样链表的长度为 $O(\sqrt{n})​$ 同时列表的长度为 $O(\sqrt{n})​$，我们可以在 $O(\sqrt{n})​$ 的时间范围内完成寻址插入和删除。 ]

  • [ 另一个需要注意的是在移动数组元素时，使用 memove 能够大大提升程序的性能 ]
