THUOJ PA1 1-2 Company

发表于 2019-01-23 | 分类于 THUOJ |

问题描述

  • 公司有n个员工，编号为1~n。
  • 员工上班时登陆管理系统登记一个code，离开时进行注销。
  • 管理人员随时想知道多少员工上班以及任一个员工登记的code。

输入输出

  • 第一行是两个整数n,m。接下来是m行，每行是以下内容之一：
    • I a c // Log In：员工 a 登录，code 为 c。若 a 已登录，则将 code 更新为 c
    • O a // Log Out：编号为 a 的员工注销。若 a 未登录，则注销无效
    • C // Close：到下班时间，所有员工注销
    • N // Number：询问有多少员工正在上班
    • Q a // Query：询问员工 a 的 code（若未登录或已注销，视为 -1）

输入输出举例：

  • 限制
    • n <= 10,000,000
    • m <= 1,000,000 1 <= a <= n code 为 [0, 2^31) 的整数

提示

一级提示

测试数据中，每种操作都可能频繁出现，或很少出现。因此，你的算法需要应对所有可能的情况。

答案会超出 int 范围，要注意各种数据类型可表示的范围，尤其是溢出问题。可选用 long long int 或 int64_t 等类型。注意各种数据类型的正确输入、输出方式。

  • Bitmap / 常数时间初始化的数组——讲义 02XA
  • Bitmap: 习题解析 [2-34]

二级提示

  • 避免频繁初始化整个数组。
  • 可以借助Bitmap记录数组的各元素是否已被初始化。将一个（或所有）元素置为未初始化状态，实质上只需要打破“校验环”两个验证条件中的任何一个，考虑如何打破校验条件的复杂度最低。
  • 另一种解题思路是，数组初始化 1 次后，记录改变了的元素，今后不再初始化整个数组，只初始化改变了的元素。

用 scanf("%c", ...) 或 getchar() 读入一个字符时，可能会读到空格、制表符、回车符或换行符，尽量避免使用这个方法。使用 scanf("%s", ...) 会自动过滤空白字符。

Windows 和 Linux 上换行符有 "\r\n" 跟 "\n" 的差别，你的程序最好有一定鲁棒性，能处理两种情况。

思路

  • 先实现低效版，再思考改进。

低效版

  • 先写一个低效版。其中通过while (isspace(op)) scanf("%c", &op);来滤去空格以及换行制表等符号。
  • 算法低效之处主要在于close()函数，当下班时只能对所有的元素进行赋值-1，而不能仅仅只针对已上班的员工。其他的主要函数时间复杂度都是O(1).
  • 总的时间复杂度为O(n)，空间复杂都为O(n)
  • 除了9,10,19测试点超时，其余测试点均可以通过。
  • 将此算法生成的执行文件作为对拍的标准程序。

思考改进

  • 考虑加上习题解析[2-34]中bitmap的思路
  • 采用同时支持set() clear()这两样操作的类，参考代码x2.9。
  • 可以在O(1)时间隐式初始化（但是还有至少要有O(n)的分配内存的时间的）
  • 查上班人数时还是沿用低效版的思路，用一个变量表示上班人数，log_in时加一，log_out时减一，都下班时清零。
  • 仿照bitmap设置两个向量，F[]来记录[k]被标记的次序，向量T[]来记录依次被标记的值。
  • clear()时的改进：只需将top置零，时间复杂度O(1)
  • 同时，code[]也不用全部初始化，只需每次修改code[F[k]]即可
  • 如此，所有的操作时间复杂度都是O(1)

THUOJ PA1 1-1 Team

发表于 2019-01-23 | 分类于 THUOJ |

问题描述

  • 教练ABC从编号为1-n的队员中选队员，根据他们各自的喜好来进行排序

  • a,b,c三个数组表示三个教练对队员的喜好程度

  • 每个数组为1-n的一个排列

  • 喜好程度从高到低：a[0],a[1]...a[n-1]

  • 分配规则：

    • 从未分配队员中找A最喜欢的队员分之
    • 然后再找B,C的

  • 问队员k分给了哪个教练？

  • 限制：

    • 1 <= n <= 500,000
    • 1 <= k <= n

工具

还提供了一个对拍工具 其中：

  • gen.py 随机生成测试数据的程序。
  • main.exe 一个待验证的程序，正确性未知。
  • std.py 一个蛮力的python代码。
  • test.cmd 自我检测运行文件。 做了一些整理，放github上了：https://github.com/WineChord/OJ-Auxiliary

提示

  • 大体上，1 sec内，O(n)的算法可以通过 n = 10,000,000 规模的数据，O(nlogn) 通过 500,000 规模，O(n^2) 通过 5,000 规模。
  • 本题等一些复杂度是 O(n) 的题目受限于 scanf(“%d”, ...) 的读入速度，但又不希望通过读取二进制文件等不直观的方式增加同学们的负担。
  • “懒删除”策略：从序列中移除一个元素时，可以只标记它，下次读取时跳过它，而不需要将它后面的元素都向前挪一位。思考如何做标记，能在每个教练员选队员时，快速地知道该队员是否已被选走。
  • Tips:清零或赋值时，memset 比手动赋值快几倍。输入数据时，有时 scanf 比 cin 快一些。关于读写速度，请参考 Tutorial Collection #1 5.大数据(Big)。

思路

  • 用一个数组s[n]来表示队员是否已被选走，被选走则为1，初始值为0。
  • 用i,j,m来表示数组a,b,c正在处理的喜好下标。
  • 然后按顺序访问a,b,c三个数组，每次每个数组能够成功标记一个队员

  • 对应C++ (直译版) test result

883. Projection Area of 3D Shapes-Math-2

发表于 2018-11-04 | 分类于 LeetCode |

问题描述

leetcode 883 将一个由小方块组成的物体放置在一个三维空间xyz里，考察这个物体在xy，yz，zx面上的投影。要求计算这三个投影的总面积。 输入 $v=grid[i][j]$ 表示 $(i,j)$ 处的高度为 $v$。

思路

首先得出 $grid$ 每一个维度的最大值，假如为 $m\times n$，并数出 $grid$ 中一共有 $k$ 个元素。则

  • 对于xy面上：

$$ S_{xy} = k $$

  • 对于yz面上，令 $h_y[j]$ 为 $y=j$ 时 $z$ 的取值，则有

$$ h_y[j] = \max_{0\le i\le m-1}(grid[i][j]) $$ 则

$$ S_{yz} = \sum_{j=0}^{n-1}h_y[j] $$

  • 对于zx面上，令 $h_x[i]$ 为 $x=i$ 时 $z$ 的取值，则有

$$ h_x[i] = \max_{0\le j \le n-1}(grid[i][j]) $$ 从而

$$ S_{zx} = \sum_{i=0}^{m-1}h_x[i] $$

实现

题解把两重循环简化成一重循环了，还有一点就是它直接认为是 $N \times N$ 的大小，所以只用了一个变量（题目中确实有说），我想复杂了，认为每一行的长度可能不同，如果题目这样出的话我觉着可能少不了一次遍历，找到列数的最大值，不过这样的话后面的循环就必须重复两次了。
