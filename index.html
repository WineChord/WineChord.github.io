THUOJ PA1 1-2 Company

发表于 2019-01-23 | 分类于 THUOJ |

问题描述

  • 公司有n个员工，编号为1~n。
  • 员工上班时登陆管理系统登记一个code，离开时进行注销。
  • 管理人员随时想知道多少员工上班以及任一个员工登记的code。

输入输出

  • 第一行是两个整数n,m。接下来是m行，每行是以下内容之一：

input:
10 8
I 1 2
I 1 4
Q 1
C
I 2 4
O 2
Q 2
N
output:
3
//“Q 1” 答案是 4，“Q 2” 答案是 -1，“N” 答案是 0，所有答案之和为 3
`

  • 限制
    • n <= 10,000,000
    • m <= 1,000,000
      1 <= a <= n
      code 为 [0, 2^31) 的整数

提示

一级提示

测试数据中，每种操作都可能频繁出现，或很少出现。因此，你的算法需要应对所有可能的情况。

答案会超出 int 范围，要注意各种数据类型可表示的范围，尤其是溢出问题。可选用 long long int 或 int64_t 等类型。注意各种数据类型的正确输入、输出方式。

  • Bitmap / 常数时间初始化的数组——讲义 02XA
  • Bitmap: 习题解析 [2-34]

二级提示

  • 避免频繁初始化整个数组。
  • 可以借助Bitmap记录数组的各元素是否已被初始化。将一个（或所有）元素置为未初始化状态，实质上只需要打破“校验环”两个验证条件中的任何一个，考虑如何打破校验条件的复杂度最低。
  • 另一种解题思路是，数组初始化 1 次后，记录改变了的元素，今后不再初始化整个数组，只初始化改变了的元素。

用 scanf(“%c”, …) 或 getchar() 读入一个字符时，可能会读到空格、制表符、回车符或换行符，尽量避免使用这个方法。使用 scanf(“%s”, …) 会自动过滤空白字符。

Windows 和 Linux 上换行符有 “\r\n” 跟 “\n” 的差别，你的程序最好有一定鲁棒性，能处理两种情况。

问题描述

  • 教练ABC从编号为1-n的队员中选队员，根据他们各自的喜好来进行排序
  • a,b,c三个数组表示三个教练对队员的喜好程度
  • 每个数组为1-n的一个排列
  • 喜好程度从高到低：a[0],a[1]…a[n-1]
  • 分配规则：
    • 从未分配队员中找A最喜欢的队员分之
    • 然后再找B,C的

  • 问队员k分给了哪个教练？

  • 限制：

    • 1 <= n <= 500,000
    • 1 <= k <= n

883. Projection Area of 3D Shapes-Math-2

发表于 2018-11-04 | 分类于 LeetCode |

问题描述

leetcode 883
将一个由小方块组成的物体放置在一个三维空间xyz里，考察这个物体在xy，yz，zx面上的投影。要求计算这三个投影的总面积。
输入 $v=grid[i][j]$ 表示 $(i,j)$ 处的高度为 $v$。

思路

首先得出 $grid$ 每一个维度的最大值，假如为 $m\times n$，并数出 $grid$ 中一共有 $k$ 个元素。则

  • 对于xy面上：

$$
S_{xy} = k
$$

  • 对于yz面上，令 $h_y[j]$ 为 $y=j$ 时 $z$ 的取值，则有

$$
h_y[j] = \max_{0\le i\le m-1}(grid[i][j])
$$
则

$$
S_{yz} = \sum_{j=0}^{n-1}h_y[j]
$$

  • 对于zx面上，令 $h_x[i]$ 为 $x=i$ 时 $z$ 的取值，则有

$$
h_x[i] = \max_{0\le j \le n-1}(grid[i][j])
$$
从而

$$
S_{zx} = \sum_{i=0}^{m-1}h_x[i]
$$

实现

题解把两重循环简化成一重循环了，还有一点就是它直接认为是 $N \times N$ 的大小，所以只用了一个变量（题目中确实有说），我想复杂了，认为每一行的长度可能不同，如果题目这样出的话我觉着可能少不了一次遍历，找到列数的最大值，不过这样的话后面的循环就必须重复两次了。
