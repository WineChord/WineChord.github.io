Ch9 Perspectives about Machine Learning

9.1 What is Machine Learning

  • 是人工智能的一个分支，从事构建和研究可以从数据中学习的系统

  • Artificial Intelligence

    • 研究感知外部环境并为某个目标采取行动的“智能体”

  • Machine Learning

    • 人工智能的一个分支，从事构建可以从数据中学习的系统

  • 与其他学科的关系

    • 统计学习
      • 取自统计学的机器学习框架
    • 模式识别
      • 识别数据中的模式（$\approx$ 机器学习+数据模式）
    • 数据挖掘
      • 发现数据中的未知特性（$\approx$ 机器学习+数据库）
    • 计算机视觉
      • 从图像中提取信息（$\approx$ 机器学习+图像处理）

  • 两种通用的学习类型

    • Inductive Learning 归纳学习（bottom-up logic）
      • 从特定的训练实例中获得或发现通用的规则或事实
    • Deductive Learning 演绎学习（top-down logic）
      • 使用一套已知的规则和事实去推导适合该训练数据的猜测

  • 机器学习的形式化定义

    • Tom Mitchell

      • 1997:

      A computer program is said to learn from experience E with respect to some class of tasks T and performance measure P, if its performance at tasks in T, as measured by P, improves with experience E.

    • Connectionism 联结主义

    • Symbolicism 符号主义
      • Association Rules, Decision tree, Random Forests
    • Behaviorism 行为主义
      • Reinforcement learning

Ch8 Classic and Real-world Planning

8.1 Planning Problems

8.1.1 What is Planning

  • 动作是智能体的一个关键部分
  • 规划意味着制定一套行动计划来达到既定的目标

8.1.2 What are Planning Problems

  • A longer definition
    • 给定现实世界中一个问题的描述：
      • 初始状态，预期目标和可能的动作
    • 规划是找到一个计划：它产生从任何初始状态到达一个目标状态的一系列动作
  • A shorter definition
    • 制定一个达到既定目标的行动计划

8.1.3 What is Classical Planning

  • 经典规划具有特征：
    • fully observable 完全可观测
    • a unique known initial state 唯一已知初始状态
    • static environments 静态环境
    • deterministic actions 确定性的动作
    • can be taken only one at a time 每次仅一个动作
    • a single agent 单一智能体

8.1.4 Planning Difficulties

actions 动作 确定的还是不确定的？是否持续一段时间？可并发执行还是串行？

initial states 初始状态 有限的还是任意多的？
objective 目标 要达到指定的目标状态？要最大化回报函数？
agents 智能体 一个还是多个？合作还是单干？
  • Problem-solving Agent vs. Planning Agent 问题求解智能体与规划智能体

8.1.5 About PDDL

  • PDDL(Planning Domain Definition Language) 规划领域定义语言是对AI规划语言标准化的一种尝试，于1998年首次开发

8.1.6 Three Components to Define a Planning Task

  • State
    • represented as a conjunction of fluents (fluents: a relation that varies from one to next) 变数的合取
  • Actions
    • described by a set of action schemas, implicitly define the function 一组动作模式
  • Goal
    • represented as conjunction of literals(literals: an elementary proposition) 命题的合取

8.2 Classic Planning

8.2.1 Planning as State-Space Search

Two approaches to searching for a plan
  • 1) Forward state-space search 前向状态空间搜索
    • 从初始状态开始
    • 运用该问题的动作
    • 朝着一个目标状态向前搜索
  • 2) Backward relevant-states search 后向状态空间搜索
    • 从表示该目标的状态集开始
    • 运用反向的动作
    • 朝着初始状态向后搜索
Heuristics for planning
  • 将搜索问题视为一个图
    • 其中结点表示状态，边为动作，寻找一条连接初始状态至某个目标状态的路径
  • 简化该问题的两种方法
    • 增加边
    • 状态抽象（state abstraction）：将多个节点组织在一起，形成具有较少状态空间抽象，从而容易搜索。
Two heuristics by adding edges to the graph
  • 1) Ignore-preconditions heuristic 忽略前提启发法
    • 放弃动作中所有的前提条件
    • 每个动作变成可作用于每个状态，并且任一目标变数可以用一个步骤实现
  • 2) Ignore-delete-lists heuristic 忽略删除表启发法
    • 从所有动作中移除删除表，即，从作用中删除所有的否定文字
    • 这样便使其可以朝着目标单调进展：任何动作都不会取消另一个动作的进展

8.2.2 Planning Graphs

  • 规划图
  • 是一个组成层次的有向图
    • 首先，初始状态的层次 $s_0$ 包含表示每个变数的结点
    • 然后，层次 $A_0$ 包含可能适用于 $S_0$ 的每个动作的结点
    • 然后，交替进入层次 $S_i$ ，接着是 $A_i$
    • 直到到达一个结束条件
  • 仅适用于命题规划问题
    • 无变量
  • 调用 EXPAND-GRAPH 来增加一层，直到通过调用 EXTRACT-SOLUTION 找到一个解，或者没有可能存在的解

8.2.3 Other Classical Planning Approaches

  • planning as Boolean satisfiability 布尔可满足性
    • 确定是否存在满足给定布尔表达式的解释的问题
      • 可满足表达式：表达式的值可为 True
      • 不可满足表达式：表达式的值始终为 False
  • planning as first-order logical deduction 一阶逻辑推理
  • planning as constraint satisfaction 约束满足
    • CSP (约束满足问题)技术对调度问题很有效
    • 可将有界规划问题进行编码为 CSP，例如找到一个长度为 $k$ 的规划的问题
    • 还可以将规划图编码为 CSP
  • planning as plan refinement 规划精进
    • Totally ordered plan 全序规划
    • Partially ordered plan 偏序规划

8.3 Planning and Scheduling

8.3.1 Planning and Scheduling

  • 规划与调度
  • 经典规划可以表示做什么，按什么顺序
  • 无法表示动作持续多长时间，什么时候发生
  • Plan First and Schedule Later
    • 将问题分为规划阶段和调度阶段
      • 规划阶段：
        • 选择具有某种有序约束的动作
        • 去满足问题的目标
      • 调度阶段
        • 在规划中增加时间信息
        • 去满足资源和期限的约束
  • 表征时间和资源约束
    • 调度问题包含一系列作业，每个作业包含一组具有顺序约束的动作
    • 每个动作有一段持续时间和一组资源约束
    • 每个资源约束指定：类型，数量，可消费或可重用
    • 动作可以产生资源
    • 解决方案选哟对每个动作指定起始时间，并且要满足所有时间顺序约束和资源约束

8.3.2 Solving Scheduling Problems

  • 要使规划持续时间最短，必须找到与排序约束一致的所有动作的最早开始时间
  • 将排序约束视为一个有向图
  • 可以将关键路径法（CPM）用于该图，来确定每个动作可能的开始与结束时间
  • 一个表示偏序计划的图的路径是一个线性排序的动作序列
  • 如果引入资源约束，所导致的开始和结束时间的约束变得更加复杂

8.4 Real-World Planning

8.4.1 Hierarchical Planning

  • 经典规划
    • 一组固定的动作
    • 问题：最新式的算法可以生成包含数千个动作的解
  • 分层规划
    • 将高层，抽象的任务分解为底层，具象的任务
    • 在层次结构的每一级，计算任务被缩减为少量活动，因此计算成本也减少
  • 基本动作 Primitive action
    • 经典规划中的动作，具有经典的前提-效用模式
    • 没有提炼过程
  • 高级动作 High-level action (HLA)
    • 层次任务网络规划中的重要概念
    • 每个 HLA 有一个或多个可能的提炼，每个动作可以是一个 HLA，或一个基本动作

8.4.2 Multi-agent Planning

  • 当环境中有多个智能体时，每个面临多智能体规划问题，试图通过其他智能体的帮助或阻碍达到自己的目标。
  • 这种多智能体规划涉及多个智能体之间协调资源和活动
  • 该主题也涉及到多个智能体在执行计划（分布式连续规划）时如何能够实时动作
  • 单智能体 & 多智能体
  • 单智能体
    • 多效用器
      • 一个智能体有多个可以并发运行的效用器，如，一个人可以同时一边打字一边说话
    • 多躯体
      • 效应器物理分解为独立的单元，但是作为一个躯体动作，例如，工厂里的传送机器人机群
  • 多智能体
    • 多智能体之间协调资源与动作
  • 多智能体的特性
    • Autonomy 自主性
    • Local views 局部视野
    • Decentralization 分散化
  • Issues in Multi-agent Planning
    • 当这些智能体具有不同目标时
    • 1) 多同步动作的表示与规划所涉及的问题
      • Actor
      • Multi-actor
      • Multiple simultaneous actions
    • 2) 真正的多智能体环境中所发生的合作，协调和竞争的问题
      • Convention 协定
      • Communication 通信
      • Plan recognition 规划认可

8.5 Decision-theoretic Planning

  • 决策理论规划
  • 经典规划是寻找一个以最小代价到达其目标的计划
  • 决策理论规划是寻找一个以最大期望效用（MEU maximum expected utility）到达其目标的规划
  • 如何形式化以及求解
    • 如何对决策理论规划问题进行格式化？
      • 马尔可夫决策过程 Markov Decision Process
    • 如何求解？
      • 动态规划 Dynamic Programming

8.5.1 Markov Decision Process

  • 是一种离散时间随机控制过程，意味着动作结果仅仅依赖于当前状态

  • 是一个五元组 $(S,A,P,R,\gamma)$ ，其中

    • $S$ 为状态集
    • $A$ 为动作集
    • $P(s’|s,a)$ 为从 $s$ 导出 $s’$ 的概率
    • $R(s,a,s’)$ 为一个回报函数
    • $\gamma\in [0,1]$ 为衰减

  • 核心问题

    • 经典规划的核心问题
      • 智能体所在的环境是确定的
      • 求解该问题是找到一个达到其目标的计划
    • 马尔可夫决策过程的核心问题
      • 处于一个离散时间随机环境
      • 求解该问题是找到一个控制其过程的策略
      • Finding policy is the core problem to solve MDPs

  • 给定一个 $MDP(S,A,P,R,\gamma)$ ，一个策略是一个计算函数 $\pi$ ，它对每个状态 $s$ 生成一个动作 $a$

    • 一个确定性策略被定义为 $\pi:S\rightarrow A$

    • 一个随机策略也可以被定义为

      • $$
        \pi:S\times A\rightarrow [0,1]\
        where\ \pi(s,a)\ge0\ and\ \sum_a\pi(s,a)=1
        $$

8.5.2 Dynamic Programming

Utilities and Optimal Policies 效用和优化策略

  • 在顺序决策问题中，偏好由状态顺序之间的顺序来表示

  • 通常采用一个累加效用函数

    • $$
      U([s_0,s_1,s_2,…])=R(s_0)+R(s_1)+R(s_2)+…=\sum_iR(s_i)
      $$

  • 一个状态（亦称真值）的效用被定义为

    • $U(s_i)$ 假设最佳动作结束之前的预期回报值的总和

  • 两个优化策略：值迭代和策略迭代

1) Value Iteration 值迭代
  • 基本思想
    • 计算每个状态的效用，然后使用该状态效用在每个状态中选择一个最佳动作
    • 不使用 $\pi$ 函数，而 $\pi$ 值在 $U(s)$ 中计算
  • 贝尔曼效用等式 Bellman equation for utilities
    • $U(s)=R(s)+\gamma\ \max_{\alpha\in A(s)}\sum_{s’}P(s’|s,a)U(s’)$
2) Policy Iteration

  • 基本思想

    • 交替执行如下两个阶段

      • Policy evaluation

        • 给定一个策略 $\pi_i$ ，如果 $\pi_i$ 被执行的话，计算每个状态的效用 $U_i$

        • $$
          U_i(s)=R(s)+\gamma\ \sum_{s’}P(s’|s,\pi_{i}(s))U_i(s’)
          $$

      • Policy improvement

        • calculate a new MEU(maximum expected uility) policy $\pi_{i+1}$, using one-step look-ahead based on $U_i$.

        • $$
          \pi^*(s)=\gamma\ argmax_{\alpha\in A(s)}\sum_{s’}P(s’|s,a)U(s’)
          $$

Ch7. Reasoning by Knowledge

7.1 Overview

7.1.1 Data, Information, Knowledge and Wisdom

7.1.2 Explicit and Tacit Knowledge 显性与隐性知识

  • Explicit knowledge 显性知识
    • 可表示为形式语言，包括语法陈述，数学表达式等
    • 可以快捷地转化为其他形式
    • 可以容易地用计算机语言，决策树和规则等表示
  • Tacit knowledge 隐性知识
    • 个人的经验和无形的因素，如观点等
    • 难以用形式化语言来表示
    • 神经网络提供了表示隐性知识的方法

7.1.4 Knowledge Base and Knowledge Base System

  • 知识库–用于区分数据库
  • 知识库被用于存储复杂的结构和非结构化知识。由一套语句组成，每个语句都是由一种被称为知识表示语言来表示的，从而表示关于世界的某些断言
  • 一个知识库系统（KBS）由知识库和推理引擎组成，其中，知识库表示关于世界的事实，推理引擎则可以基于这些事实进行推理

7.1.5 Knowledge Engineering (KE)

  • 知识工程
    • 是构建，维护和使用知识库系统中所关联的所有技术，科学和社会的方方面面

7.1.6 Knowledge-based Engineering (KBE)

  • KBE是将基于知识的系统技术用于制造设计和生产领域。

7.2 Knowledge Representation

7.2.1 Overview of Knowledge Representation

  • 什么是知识表示？
    • 关注于设计计算机表示来采集关于世界的知识，可用于解决复杂的问题
    • 与过程性代码相比，使复杂的软件容易定义和维护，可用于专家系统
  • 为什么采用知识表示
    • 传统的过程性代码并非是解决复杂问题的最好形式

7.2.2 Core Issues of Knowledge Representation

  • Primitive 原语
    • 用于表示知识的基础框架，例如：语义网络，一阶逻辑等
  • Meta-representation 元表示
    • 知识表示语言用该语言本身表示
  • Incompleteness 不完备性
    • 将确定性因子与规则和结论相关联
  • Universals vs. Facts 共性与事实
    • Universals: 关于世界的一般性描述
    • Facts: 共性中的具体事例
  • Expressive adequacy 表现的充分性
    • How expressive they intend their representation to be.
  • Reasoning efficiency 推理的有效性
    • Refer to the run time efficiency of the system

7.3.5 Formation Rules in First Order Logic

  • The formation rules define
    • terms and formulas
  • The formation rules can be used to write a formal grammar for terms and formulas.
  • Formation rules are generally context-free, i.e.,
    • each production has a single symbol on the left side.
  • Terms:
    • Rule1: Variables
      • Any variable is a term
    • Rule2: Constants
      • Any constant is also a term
    • Rule3: Functions
      • Any expression $f(t_1,…t_n)$ of $n$ arguments is a term, where each argument $t_i$ is a term, and $f$ is a function symbol of valence $n$. In particular, symbols denoting individual constants are 0-ary function symbols, and thus terms.
  • Formulas:
    • Predicate symbols: If P is an n-ary predicate symbol and $t_1,…,t_n$ are terms, then $P(t_1,…,t_n)$ is a formula.
    • 谓词符号：若 P 是一个 n 元谓词符号并且 $t_1,…,t_n$ 是项，则 $P(t_1,…,t_n)$ 是一个公式。
    • Equality: If the equality symbol is considered part of logic, and $t_1$ and $t_2$ are terms, then $t_1=t_2$ is a formula.

    • Negation: if $\varphi$ is a formula, then $\neg \varphi$ is a formula.
    • Binary connectives: If $\varphi$ and $\psi$ are formulas, then $(\varphi\implies\psi)$ is formula. Similar rules apply to other binary logical connectives.
    • Quantifiers: If $\varphi$ is a formula and $x$ is a variable, then $\forall x\varphi$ and $\exist x\varphi$ are formulas.

      • belief network 信念网络
      • probabilistic network 概率网络
      • causal network 因果网络
    • 一个贝叶斯网络表示一组随机变量和他们的条件依赖关系
      • 如：可以表示疾病与症状之间的概率关系
    • 两种观点：
      • 将该网络视为一种联合概率分布的表示
      • 将其视为一组条件独立语句的一种编码
    • 第一种观点有利于理解如何构建网络
    • 第二种观点有助于设计推理过程

7.5.5 Representing Full Joint Distribution

  • 采用条件分布元素的乘积：
    • $P(x_1,…,x_n)=\prod_{i=1}^nP(x_i|parents(X_i))$

7.5.6 Constructing Bayesian Networks

  • 首先，用条件概率公式对联合概率进行改写，使用如下乘积规则

    • $P(ab)=P(a|b)P(b)$
    • $P(x1,…,x_n)=P(x_n|x_{n-1},…,x_1)P(x_{n-1},…,x_1)$

  • 然后，重复该过程，将每个合取概率缩减为条件概率和较小的合取，最终得到一个大的乘积

    • $$
      \begin{align}
      P(x_1,…,x_n)&=P(x_n|x_{n-1},…,x_1)P(x_{n-1}|x_{n-2},…,x_1)…P(x_2|x_1)P(x_1)\
      &=\prod_{i=1}^nP(x_i|x_{i-1},…,x_1)
      \end{align      }
      $$

    • 此式为任意一组随机变量的链式法则

  • 联合分布的规格等同于一般性断言，即：对于网络中的每个变量 $X_i$有

    • $$
      P(X_i|X_{i-1},…,X1)=P(X_i|Parents(X_i))\
      provided\ that\ Parents(X_i)\sube {X_{i-1},…,X_1}
      $$

  • 该等式说明：贝叶斯网络是域的一个正确表示，仅当给定其父节点，每个节点条件独立于结点序中其他前驱结点时。

  • 可用如下方法来满足该条件

      1. 结点：先确定要对域建模的变量集，${X_,…,X_n}$
      2. Links: For i = 1 to n do:
        • Choose from $X_1,…,X_{i-1}$, aminimal set of parents for $X_i$.
        • For each parent insert a link from the parent to $X_i$
        • Write down the conditional probability table (CPT)
          • $P(X_i|Parents(X_i))$
    • 结点 $X_i$ 的父辈应该包含直接影响 $X_i$ 的 $X_1,…,X_{i-1}$ 的所有节点。

7.5.7 Compactness

  • 贝叶斯网络远比全联合分布紧凑。它的紧凑性是局部结构化（也称稀疏）系统的一般特性。
  • 在局部结构化系统中，每个子成分仅直接与其他成分的有限数量打交道
  • 贝叶斯网络中，具有 $k$ 个父节点的布尔变量的 CPT (条件概率表) 有 $2^k$ 行
  • 假设有 $n$ 个布尔变量
    • Bayesian networks: 贝叶斯网络
      • 如果每个变量的父节点不超过 $k$ 个，则全部网络可以用 $n2^k$ 个数指定
    • Full joint distribution: 全联合分布
      • 包含 $2^n$ 个数

7.5.8 Node Ordering

  • 只有当选择一个好的结点排序，我们才会得到一个紧凑的贝叶斯网络

7.5.9 Conditional Independence Relations

  • Numerical semantics: 数值语义
    • 一个结点，给定其父节点后，条件独立于其他前驱结点
      • $P(x_1,…,x_n)=\prod_{i=1}^nP(x_i|parents(X_i))$
  • Topological semantics: 拓扑语义
    • 每个节点，给定其父节点后，条件独立于它的非后继结点
  • Nemerical semantics $\Leftrightarrow$ Topological semantics
  • Markov blanket 马尔可夫覆盖
    • 一个结点，给定其马尔可夫覆盖后，条件独立于网络中的所有其他结点（即：父节点，子节点，以及子节点的其他父节点）

6.1.1 What is CSP?

  • 是数学问题，定义为其状态必须满足若干约束和限制的一组对象

6.1.2 Standard Search vs. Constraint Satisfaction Problem

  • Standard search problem 标准搜索问题
    • 其状态是原子的或不可分割的，一个没有内部结构的黑盒子
  • Constraint satisfaction problem 约束满足问题
    • 状态用因子表示，是一系列变量，每个有相应的值
    • 发挥状态结构的长处
    • 采用一般用途而不是问题特有的启发式

6.1.3 Why Study CSP?

  • CSP 通常呈现出高复杂性，需要将启发式与组合搜索方法相结合
  • 某些形式的 CSP
    • Boolean Satisfiability Problem 布尔可满足性问题

    • Answer Set Programming 答案集编排

    • Ctyptarithmetic 密码算术
    • Sudoku 数独

6.1.4 Formal Def of CSP

  • A CSP is defined as a triple <X,D,C>
    • X is a set of variables, ${X_1,…,X_n}$
    • D is a set of domains, ${ D_1, …, D_n}$. One $D_i$ for each variable $X_i$ , consisting of a set of values ${V_1,…,V_k}$
    • C is a set of constraints, ${ C_1,…,C_m}$. Each $C_i$ consists of a pair <scope,rel> where, scope is a tuple of variables that participate in the constraint, rel is a relation that defines the values that those variables can take on.
  • To solve a CSP, we need to define a state space and the notion of a solution.
  • Each state is defined by assignment of values to variables: ${X_i=v_i,X_j=v_j,…}$.
    • Consistent assignment 一致性赋值
      • 一种不违反任何约束的合法赋值
    • Complete assignment 完整性赋值
      • 每个变量都被赋值，并且该赋值是一致的，完整的
    • Partial assignment 局部性赋值
      • 仅对某些变量进行赋值

6.2 Constraint Propagation: Inference in CSPs

6.2.1 Overview of Constraint Propagation

  • Regular state-space search
    • can do only one thing: search
  • CSPs:
    • can do search, choose a new variable assignment from several possibilities
    • can also do a specific type of inference, called constraint propagation
  • Constraint Propagation
    • uses the constraints to reduce the number of legal values for a variable, which in turn can reduce the legal values for another variable, and so on.
  • Those techniques are used to modify a constraint satisfaction problem
  • More precisely, they enforce a form of local consistency, which are conditions related to the consistency of a group of variables and/or constraints
  • Constraint propagation has various uses:
    • 1) turns a problem into one that is equivalent but is usually simpler to solve,
    • 2) may prove satisfiability/unsatisfiability of problems
  • This always happens for some certain kinds of problems.
  • There are different types of local consistency:
    • Node consistency 结点一致性
    • Arc consistency 弧一致性
    • Path consistency 路径一致性
    • k-consistency k 一致性
  • Most popular method is AC-3 algorithm which enforces arc consistency

6.2.2 Node Consistency

  • Def: A single variable(node) is node-consistent, if all the values in the variable’s domain satisfy the variable’s unary constraints.

6.2.3 Arc Consistency

  • Def: A variable is arc-consistent, if every value in its domain satisfies the variable’s binary constraints.

6.2.4 Path Consistency

  • Def: A two-variable set ${X_i,X_j}$ is path-consistent with respect to a third variable $X_m$ if, for every assignment ${X_i=a,X_j=b}$ consistent with the constraints on ${X_i,X_j}$, there is an assignment to $X_m$ that satisfies the constraints on ${X_i,X_m}$ and ${X_m,X_j}$.
  • Why called path consistency
    • because it as looking at a path from $X_i$ to $X_j$ with $X_m$ in the middle.

6.2.5 k-Consistency

  • Def: A CSP is k-consistent if, for any set of k-1 variables and for any consistent assignment to those variables, a consistent value can always be assigned to any $k^{th}$ variable.
  • The notion of k-consistency is the stronger forms of propagation.
  • Features of k-consistency
    • k=1: same as node consistency
    • k=2: same as arc consistency
    • k=3: same as path consistency

      • then MAX’s moves in the states resulting from every possible response by MIN, and so on.

Minimax Theorem

For every two-player, zero-sum game with finitely many strategies, there exists a value V and a mixed strategy for each player, s.t.

(a) Given player 2’s strategy, the best payoff possible for player 1 is V

(b) Given player 1’s strategy, the best payoff possible for player 2 is -V

  • 对于零和博弈来说，其名称 minimax 由来是因为每个玩家会使对手可能的最大收益变得最小，还会使自己的最大损失变得最小。

5.3 Alpha-Beta Pruning

Overview

  • The problem with minimax search:
    • Number of game states is exponential in depth of the tree.
    • We can’t eliminate the exponent, but we can effectively cut it in half.
  • The trick to solve the problem:
    • Compute correct minimax decision without looking at every node in game tree.
    • That is, use “pruning” to eliminate large parts of the tree
  • What is alpha-beta pruning?
    • It is a search algorithm that seeks to decrease the number of nodes that are evaluated by the minimax algorithm

Why Called Alpha-Beta

  • Alpha-beta pruning gets its name from the following two parameters:
    • $\alpha$ : 沿着 MAX 路径上的任意选择点，迄今为止我们已经发现的最高值
    • $\beta$ : 沿着 MIN 路径上的任意选择点，迄今为止我们已经发现的最低值
  • Alpha-beta search respectively:
    • updates the values of $\alpha$ and $\beta$ as it goes along, and
    • prunes the remaining branches at a node as soon as the value of the current node is known to be worse than the current $\alpha$ or $\beta$ value for MAX or MIN.

  • 采用机器学习和树搜索技术相结合的方式，并且用人类和计算机走棋的棋局进行大量的训练

  • 树搜索
    • 蒙特卡洛树搜索
  • 强化学习
    • 用于改善其走棋

Monte-Carlo Tree Search in AlphaGo

  • (a) Selection: Each simulation traverses the tree by selecting edge with maximum action value Q + bonus u(P) that depends on a stored prior probability P for that edge.
  • (b) Expansion: The leaf node may be expanded; the new node is processed once by the policy network $p_\sigma$ and the output probabilities are stored as prior probabilities P for each action.
  • (c) Evaluation(Simulation): The leaf node is evaluated in two ways: 1) using the value network $v_\theta$; 2) by running a rollout to the end of the game with the fast rollout policy $p_\pi$ , then computing the winner with function r
  • (d) Backup(Back propagation): Action values Q are updated to track the mean value of all evaluations $r(\cdot)$ and $v_\theta(\cdot)$ in the subtree below that action.

Neural Network Training Pipeline in AlphaGo

  • A fast rollout policy $p_\pi$ and supervised learning(SL) policy network $p_\sigma$ are trained to predict human expert moves in a data set of positions
  • A reinforcement learning(RL) policy network $p_\rho$ is initialized to the SL policy network, and then improved by policy gradient learning to maximize the outcome
  • A new data set is generated by playing games of self-play with the RL policy network
  • A value network $v_\theta$ is trained by regression to predict the expected outcome

Neural Network Architecture in AlphaGo

  • The policy network 策略网络

    • input: the board position
    • passes s through convolutional layers with parameters $\sigma$ or $\rho$
    • outputs: a prbability distribution over legal moves a.

  • The value network 价值网络

    • input: the board position s’
    • similarly uses many convolutional layers with parameters $\theta$
    • output: a scalar value $v_\theta(s’)$ that predicts the expected outcome.

4.2.2 Local Beam Search

  • 局部束搜索
  • Keeping just one node in memory might seem to be an extreme reaction to the problem of memory limitations.
  • Local beam search keeps track of k states rather than just 1
    • It begins with k randomly generated states
    • At each step, all the successors of all k states are generated.
    • If any one is a goal, the algorithm halts, else it selects the k best successors from the complete list, and repeats.
  • Variant of LBS
    • Stochastic Beam Search 随机束搜索
      • 局部束搜索会很快地集中在状态空间的某个小区域内，使得搜索代价比爬山法还要昂贵
      • 模仿随机爬山法，有助于缓解这个问题
      • Instead of choosing best k successors, it chooses k successors randomly, with the probability of choosing a successor being an increasing function of its value.
      • Stochastic beam search bears some similar to the process of natural selection.

    • 是一种基于目标的智能体，通过搜索来解决问题

3.1.2 Algorithm of Simple Problem Solving Agents

  • State space 状态空间，可形式化地定义：
    • 初始状态
    • 动作
    • 转换模型
  • Graph
    • 状态空间形成一个图，其中结点表示状态，链接表示动作
  • Path
    • 状态空间的一条路径是由一系列动作连接的一个状态序列

3.1.4 Five Items to Formulate a Problem

  • Actions 动作

  • Goal test 目标测试

3.2 Example Problems

3.2.1 Example1: Vacuum-cleaner world

  • 真空吸尘器的世界

    • 一个仅仅有两个地点的真空吸尘器世界：方形区域A和B

    • 该吸尘器智能体感知它在哪个方形区域，以及该方形区域是否有灰尘。它可以选择左行，右行，吸尘，或者空操作，即：

      • Percepts: 感知：位置和内容
        • location and contents, e.g. [A, Dirty]
      • Actions: 动作：左/右行，吸尘，空操作
        • Left, Right, Suck, NoOp

    • 正确的功能是什么？

    • 是否可用一个小智能体程序实现？

  • States 状态

    • 智能体在两个地点中的一个，每个也许有灰尘或者没有
    • 可能的状态，两个地点：$2\times 2^2=8(n\times 2^n)$

  • 1）Initial state 初始状态：可为任何状态

  • 2）Actions 动作：每个状态仅有三个动作：左/右移，吸尘

  • 3）Transition model 转换模型：

    • 除了以下动作，其余都有
      • 在最左边左移
      • 在最右边右移
      • 在情节区域进行吸尘

  • 4）Goal test 目标测试

    • 是否所有的区域内都干净

  • 5）Path cost 路径代价

    • 等于路径的步数

3.2.2 Example2: 8-puzzle

  • $3\times 3$ 棋盘上有 $8$ 个数字棋子和一个空格
  • 与空格相邻的滑块可以移向该空格，目的是达到一个指定的目标状态
  • States 状态
    • $8$ 个数字滑块每个占据一个方格，而空格则位于最后一个方格
  • 1）Initial state 初始状态：可为任意状态
  • 2）Actions 动作
    • 上下左右
    • 不同的子集依赖于空格的位置
  • 3）Transition model 转换模型
    • 给定状态和动作，其返回结果状态。如：
      • 若对初始状态施加左移操作，由此产生的状态则使 $5$ 与空格互换
  • 4）Goal test 目标测试
    • 即检查状态是否与目标状态相符
  • 5）Path cost 路径代价：等于路径的步数
  • 该问题属于滑块难题（Sliding block puzzles），被认为是NP完全的。

3.2.3 Example3: 8-queens problem

  • 目标是将 $8$ 个皇后摆放在国际象棋的棋盘上，使得皇后之间不发生攻击（即每两个皇后不在同一行或同一列或同一对角线）
  • 两种主要的形式化方法
    • Incremental formulation 增量形式化
      • 从空状态开始，然后每次添加一个皇后改变其状态
    • Complete-state formulation 全态形式化
      • 初始时 $8$ 个皇后都放在棋盘上，然后再将她们移开
  • The incremental formulation 增量式形式化
    • State 状态
      • 第一个至第八个皇后在棋盘上任意摆放，为一个状态
    • 1）Initial state: 棋盘上没有皇后
    • 2）Actions: 添加一个皇后至任意一个空格
    • 3）Transition model: 将一个皇后添加到指定空格，再返回该棋局
    • 4）Goal test: $8$ 个皇后都在棋盘上，并且没有攻击
    • 5）Path cost: 等于步数

3.3 Searching for Solutions

3.3.1 Shortest Path Problem by Graph Search

A General Graph-search Algorithm 一种通用的图搜索算法

  • The explored(aka closed list) is an data structure to remember every expanded node.
  • The nodes in the explored or the frontier can be discarded.

3.3.2 Shortest Path Problem by Tree Search

A General Tree-search Algorithm 一种通用的树搜索算法

  • The process of expanding nodes on the frontier continues until either a solution is found or there are no more states to expand.

3.4 Uniformed Search Strategies

  • 无信息搜索
    • 又称盲目搜索
    • 意味着该搜索策略没有超出问题定义提供的状态之外的附加信息
    • 所有能做的就是生成后继节点，并且区分出一个目标状态或一个非目标状态
    • 所有的搜索策略是由结点扩展的顺序加以区分的
    • 分类：
      • 宽度优先
      • 广度优先
      • 一致代价搜索
  • 评价
    • Completeness 完备性：是否总能找到一个存在的解？
    • Time complexity 时间复杂度：花费多少时间？
    • Space complexity 空间复杂度：需要多少内存？
    • Optimal 最优性
    • 衡量时空复杂度：
      • b: 搜索树的最大分支因子
      • d: 最浅解的深度
      • m: 搜索树的最大深度

3.4.1 Breadth-first Search

  • 搜索策略：扩展最浅的未扩展结点

  • 实现：使用FIFO队列，即新的后继结点放在后面

  • 图的宽度优先搜索算法

  • Time complexity: $b+b^2+b^3+…+b^d=O(b^d)$

  • Space complexity: $O(b^d)$

  • 其中 $b$ 为分支因子，$d$ 为最浅解的深度

  • $BFS$ 不能解决指数复杂性的问题，小的分支因子除外。

3.4.2 Uniform-cost Search

  • 一致代价搜索
  • 搜索策略：扩展最低代价的未扩展结点
  • 实现：队列，按路径代价排序，最低优先
  • Time: $O(b^{1+\lfloor C^*/\epsilon\rfloor})$
  • Space: $O(b^{1+\lfloor C^*/\epsilon\rfloor})$
  • $b$ 为分支因子，$C^*$ 为最优解的代价，$\epsilon$ 为每个动作的最小代价

3.4.3 Depth-first Search

  • 搜索策略：扩展最深的未扩展结点。
  • 实现：使用LIFO队列，把后继结点放在队列的前端。
  • Time: $O(b^m)$
  • Space: $O(bm)$
  • $b$ : 分支因子，$m$ 为任一结点的最大深度

3.4.4 Variants of Depth-first Search

1）Depth-limited Search 深度受限搜索
  • 若状态状态空间无限，深度优先搜索就会发生失败
  • 可以用一个预定的深度限制 $l$ 得到解决，即：深度 $l$ 以外的结点被视为没有后继结点
  • 缺点：
    • 如果选择 $l\lt d$ ，即最浅的目标在深度限制之外，这种方法就会出现额外的不完备性
    • 如果选择 $l\gt d$，深度受限搜索也将是非最优的。
2）Iterative Deepening Search 迭代加深搜索
  • 将深度优先和宽度优先的优势相结合，逐步增加深度限制反复运行直到找到目标。
  • 以深度优先搜索相同的顺序访问搜索树的节点，但先访问的结点的累积顺序实际是宽度优先。

3.4.5 Bidirectional Search

  • 同时进行两个搜索：
    • 一个从初始状态向前搜索
    • 一个从目标向后搜索
    • 当两者相遇时停止
  • 可以通过一种剩余距离的启发式估计来导向

3.4.6 Comparing Uninformed Search Strategies

  • 无信息树搜索策略评价
Criterion Breadth First Uniform Cost Depth First Depth Limited Iterative Deepening Bidirectional
Complete $Yes^a$ $Yes^{a,b}$ $No$ $No$ $Yes^a$ $Yes^{a,d}$

  • d – if both directions use breadth-first search

3.5 Informed Search Strategies

  • 有信息搜索，又称启发式搜索
  • 采用超出问题本身定义的，问题特有的知识，因此能够找到比无信息搜索更有效的解。
  • 一般使用如下函数的一个或两者
    • 评价函数（evaluation function）：$f(n)$ ，用于选择一个结点进行扩展
    • 启发式函数（heuristic function）：$h(n)$，作为 $f$ 的一个组成部分

3.5.1 Best-first Search

  • 搜索策略
    • 一个结点被选择进行扩展是基于一个评价函数，$f(n)$
    • 大多数的最佳优先算法还包含一个启发式函数，$h(n)$
  • 实现
    • 与一致代价搜索相同
    • 然而，最佳优先搜索使用 $f(n)$ 代替 $g(n)$ 来整理优先队列
  • 启发式函数（Heuristic function）
    • $h(n)=$ 从结点 $n$ 到目标状态的最低路径估计代价
  • 特例
    • Greedy Search
    • $A^*$ Search

3.5.2 Greedy Search

  • 搜索策略
    • 试图扩展最接近目标的结点
  • 评价函数：$f(n)=h(n)$
    • 仅使用启发式函数对结点进行评价
    • $h(n)$ – 从 $n$ 到最接近目标的估计代价
  • Why call “greedy”?
    • 每一步它都试图得到能够最接近目标的结点
  • 特性：
    • Worst-case time: $O(b^m)$
    • Space complexity: $O(b^m)$
    • b: 分支因子，m: 搜索空间的最大深度

3.5.3 $A^*$ Search

  • 搜索策略
    • 避免扩展代价高的路径，使总的估计求解代价最小化
  • 评价函数
    • $g(n)$ 到达该结点的代价
    • $h(n)$ 从该节点到目标的估计代价
    • $f(n)=g(n)+h(n)$
  • Theorem: $A^*$ search is optimal

3.5.4 Iterative Deepening $A^*$ Search

  • 迭代加深 $A^*$ 搜索
  • 是迭代加深深度优先搜索的变种
    • 从 $A^*$ 搜索算法中借鉴了这一思想，即使用启发式函数来评价到达目标的剩余代价
  • 因为是一种深度优先搜索算法，内存使用率低于 $A^*$ 算法
    • 但是，不同于标准的迭代加深搜索，它集中于探索最有希望的结点，因此不会去搜索树任何同样深度的地方。
  • Comparing Iterative Deepening Search 迭代加深搜索之比较
    • Iterative deepening depth-first search
      • 使用搜索深度作为每次迭代的截止值
    • Iterative Deepening $A^*$ Search
      • 使用信息更丰富的评价函数，即 $f(n)=g(n)+h(n)$
      • $g(n)$ 为到达该节点的代价
      • $h(n)$ 为该节点到目标的估计代价

3.6 Heuristic Functions

3.6.1 Heuristics for 8-puzzle

  • 要用 $A^*$ 算法找到最短距离的解，需要一个启发式函数，通常有两个候选
    • $h_1=number\ of\ misplaced\ tiles =8$
    • $h_2 = total\ Manhattan\ distance(tiles\ from\ desired\ locations)=18$

3.6.2 Search Cost

  • 若对于所有的 $n$，$h_2(n)\ge h_1(n)$ ，则 $h_2$ 优于 $h_1$ ，因而 $h_2$ 更适合搜索。

人工智能原理 Part I. Basics

Ch1. Introduction

1.1 Overview of Artificial Intelligence

1.1.1 What is Artificial Intelligence(AI)

  • 机器或软件所展现的智能
  • 学术研究领域的名称
  • 如何创建计算机和计算机软件使之具有智能行为
  • 诞生：1956年的“达特茅斯夏季人工智能研究计划”会议

1.1.2 Turing Test

  • 图灵测试由艾伦图灵在1950年发表的“计算机器与智能”论文中提出的。
  • 旨在提供一种令人满意的关于智能的可操作定义。
  • 若一个人类提问官，在提出一些书面问题之后，无法分辨这些书面回答究竟是来自于人还是一台计算机，则认为计算机通过了该测试。

1.1.3 Visual Turing Test

  • 视觉图灵测试：2014年由 Donald Geman 提出
  • 是采用一个操作员辅助设备，根据给定的图像产生随机的二元问题序列
  • 目前的计算机视觉系统是测试任务的精度。这些任务包括对象检测，图像分割和定位。
  • 受人类理解图像能力的启发而提出的。

1.1.4 Chinese Room

  • 思想实验：中文屋，又称 “Sherle’s Chinese Room Argument”
  • 最早由美国哲学家 John Searle 1980 年在《行为与大脑科学》上发表的论文《智力，头脑与规划》提出的。
  • 试图揭示计算机绝不能描述为有“智力”(mind)或“知性”(understanding)，不管它多么智能。
  • 设想在一个房间内操作一套计算机程序来应付从门缝中塞进来的中文字符，操作人对中文一窍不通，但是他可以通过处理符号和数字生成了合适的中文字符串，从而让屋外的人认为屋内有一个精通中文的人。
  • 结论是：按程序运行的计算机可以使它看起来理解了语言，但并没有产生真正的理解。
  • 由此他断定，图灵测试的结论是不充分的。

1.2 Foundations of Artificial Intelligence

1.2.1 Mathematics

1) Logic 逻辑学 – 得出正确结论的形式规则是什么？
  • 1847 乔治布尔(George Boole)：命题逻辑(propositional logic)，又称布尔逻辑(Boolean logic)
  • 1879 戈特洛布·弗雷格(Gottlob Frege)：一阶逻辑(first order logic)，是布尔逻辑的扩展，增加了对象和关系(objects and relations)。
  • 阿尔弗雷德·塔斯基(Alfred Tarski 1902-1983)：指称理论(theory of reference)，揭示如何将逻辑中的对象与对象相关联。
2） Computation 计算 – 什么是可计算的？

  • 艾伦·图灵(Alan Turing 1912-1954)：试图精确地描述哪些函数是可计算的。

  • 科伯姆与埃德蒙兹(mid-1960s)：提出计算的易处理的概念。

  • 1972 斯蒂文·库克与理查·德卡普：提出NP完全性的理论。

    • NP-completeness:
      • P: Polynomial time 多项式时间
      • NP: Non-deterministic Polynomial time 不确定性多项式时间
      • NP-complete: both in NP and NP-hard NP和NP难的交集
    • NP 是否等于 P 还是一个悬而未解的问题。
3） Probability 概率 – 如何根据不确定信息进行推理？
  • 卡尔达诺(Cardano 1501-1576)：构建了概率的概念，将其描述为博弈事件中可能的结果。
  • 伯努利，拉普拉斯推进该理论，并引入了新的统计学方法。
  • 贝叶斯：提出了贝叶斯规则，使之成为不确定性推理的最现代方法。

1.2.2 Neuroscience

  • 神经科学：大脑如何处理信息？
    • 神经科学研究神经系统，尤其是大脑
    • 大脑在理性决策方面非常优越（但并非完美无缺）
    • 大脑不像软件那样模块化
    • 预测和仿真时决策的关键
    • 计算机与大脑的对比：
      • 大脑中神经元的数量基本是固定的，
      • 计算机中处理单元的数量，每五年增加十倍。

1.2.3 Cognitive Psychology

  • 认知心理学：人类如何思考和行动
    • 把大脑看作是信息处理设备，是研究心智过程的学科
    • 包含：
      • attention 注意机制：意识集中在某个有用感知信息子集的状态。
      • language use 语言运用：研究语言习得，语言形成的组件，语言使用时用的预期，或者许多其他相关领域。
      • memory 记忆：过程记忆，语义记忆，情景记忆。
      • perception 感知：物理感知（视觉，嗅觉，味觉，知觉）及其认知过程。
      • metacognition 元认知：是“关于认知的认知”。包裹关于认知的知识以及认知的调节。
      • problem solving 问题求解
      • creativity 创造力
      • thinking 思考
  • 认知心理学与认知科学的对比
    • 认知心理学
      • 通常通过人类参与者的心理实验来收集信息
      • 目的是研究人脑如何接受外部世界的输入，如何处理以及作用等
    • 认知科学
      • 关注于通过研究收集数据
      • 涉猎心理学，语言学，人类学，神经科学，社会学和教育学，尤其是人工智能

1.2.4 Control Theory and Cybernetics

控制理论与控制论：机器如何能在其自身的控制下运行？

  • Control theory 控制理论
    • 工程与数学的交叉学科分支
    • 处理动态系统对输入的行为，以及该行为如何通过反馈进行调整
  • Cybernetics 控制论
    • 跨学科的研究途径，探索调控系统，它们的结构，约束和可能性
    • 1948年被定义为“研究动物与机器的控制与通信的科学”
    • 21世纪，简单将其解释为“用技术控制任何系统”

1.3 History of Artificial Intelligence

1950-1956 诞生

  • 1950，艾伦图灵提出图灵测试，将其作为机器智能的度量
  • 1956，在达特茅斯学院的会议上，人工智能正式诞生

  • 1958，赫伯特·西蒙(Herbert Simon)和艾伦·纽厄尔(Allen Newell)演示了第一个AI程序，名称为逻辑理论家(Logic Theorist)
  • 1958，约翰·麦卡锡(John McCarthy)发明了著名的Lisp编程语言
  • 1960s，马斯特曼(M.Masterman)和他的剑桥大学同事设计了语义网络，用于机器翻译
  • 1963，伦纳德·武赫(Leonard Uhr)和查尔斯·瓦斯勒(Charles Vossler)发表了关于模式识别的论文，描述了第一个机器学习程序。
  • 1965，费根鲍姆(E.Feigenbaum)开创了Dendral，一个推断有机化合物分子结构的软件。这是首套专家系统。

2.4 Intelligent Agent Structure

The Structure of Agents

  • 一个智能体的行为可以在数学上被描述为一个智能体函数，将每个感知映射为动作
    $$
    f:\mathcal{P}^{\star}\rightarrow \mathcal{A}\ \ \ \ where\ \mathcal{P}^{\star}=\sum^T_{t=1}|\mathcal{P}|^t
    $$

  • $$
    \begin{align}
    Agent&=platform+agent\ program\
    platform&=computing\ device+sensors+actuators\
    agent\ program &\supset agent\ function
    \end{align    }
    $$

  • 智能体层次

    • 通常包含多个子智能体
    • 子智能体处理和执行较低级的功能
    • 智能体和子智能体构建一个完整的系统

Agent Function

  • 智能体函数是一个抽象的概念，可包括各种决策制定的原则
    • 单个选项的效用计算
    • 贯穿逻辑规则的推论
    • 模糊逻辑
    • 查找表

Agent Programs

  • 实现一个智能体功能，将感受器的输入作为当前的感知，然后返回一个动作给执行器。

Three ways to represent states for an agent

  • Atomic 原子式：每个状态是黑盒，无内部结构
  • Factored 因子式：每个状态由一组固定的属性和值组成
  • Structured 结构式：每个状态包含对象，每个具有属性以及与其他对象的关系

2.5 Category of Intelligent Agents

  • 基于感知的智力和能力的程度进行分类

2.5.1 Simple Reflex Agents

  • 仅在当前感知的基础上动作，忽略其余的感知历史

  • 基于条件动作规则：if 条件 then 动作

  • About Simple Reflex Agents

    • 仅当外部环境为完全可观测时，该智能体的功能才能发挥
    • 某些反射智能体也可以包含关于当前状态的信息，允许它们忽视执行器已被触发的条件
    • 智能体在部分可观测环境下运行时，无限循环往往是无法避免的
    • 注意：如果智能体可以随机产生其动作，有可能从无限循环中摆脱出来

  • Algorithm of Simple Reflex Agents

2.5.2 Model-based Reflex Agents

  • 一个基于模型的反射智能体可以处理部分可观测环境

  • 其当前状态存储在智能体中，维护某种结构，它描述不可见外部环境的一部分

  • About Model-based Reflex Agents

    • 关于“外部环境如何运作”的知识被称为一个外部环境模型
    • 基于模型的反射智能体将保持某种内部模型
    • 内部模型依赖于感知的历史，因此至少反射某些当前状态无法观测的方面
    • 然后它作为反射智能体以某种方式选择动作

  • Algorithm of Model-based Reflex Agents

2.5.3 Goal-based Agents

  • 通过利用“目标信息”，基于目标的智能体进一步扩展了基于模型的智能体的功能
  • About Goal-based Agents
    • 目标信息描述所希望的情形
    • 它允许智能体在多个可能性之间选择一种方式，挑选出达到目标状态的那一个
    • 搜索和规划是人工智能的子领域，致力于发现达到智能体目标的动作序列
    • 在某些情况下，基于目标的智能体似乎不太有效
    • 但它更灵活，因为这种支持其决策的知识明显地展示出来，并且可以被修改

2.5.4 Utility-based Agents

  • 一个特殊的状态可通过一个效用函数得到，该函数将一个状态映射到一个该状态效用的度量。
  • About Utility-based Agents
    • 一种更通用的性能度量，应该根据他们使得智能体多么“高兴”的程度，允许对不同的外部环境状态进行比较
    • 效用用于描述智能体的快乐程度
    • 一个理性的基于效用的智能体选择动作，将动作结果的期待效用最大化
    • 一个基于效用的智能体需要建模并记录环境，任务的轨迹，这涉及大量的感知，表征，推理，和学习的研究

2.5.5 Learning Agents

  • 学习允许智能体最初在未知的环境中运行，并且与其最初的知识相比，会变得越来越胜任。
