THUOJ PA1 Lighthouse

Posted on 2018-08-18 | In THUOJ

Description

If another lighthouse is in grey area, they can beacon each other.

Input

  • First line: the total number of lighthouse N
  • Second to (N+1)th line: lighthouse is on the point (X, Y)

Output

How many pairs of lighthouses can beacon each other.

Restrictions

  • 90% : 1 <= n <= 3 * 10^5
  • 95% : 1 <= n <= 10^6
  • All : 1 <= n <= 4 * 10^6
  • For every lighthouses, X coordinates won’t be the same, Y coordinates won’t be the same.
  • 1 <= x, y <= 10^8
  • Time: 2s
  • Memory: 256 MB

Example

  • input

  • output

Idea

It is trivial to think that use two for loops can solve the problem. However the time consumption will be too large. First construct a point struct. Use quick sort to make the point is sorted according to it X coordinate. Then calculate the inversions in the Y coordinates. The total inversions mean that they cannot form lighthouses that beacon each other. Thus the result equals to the total permutation number minus the total inversions.
The hard point is how to calculate inversions. Here uses a variation of merge sort. When doing mergesort to their Y coordinates, keep a count value to record inversions.

THUOJ PA1 Zuma

Posted on 2018-08-18 | In THUOJ

Description

Zuma is a game that has a sequence of beads. All the beads are colored but no three adjacent ones are allowed to be with a same color. Now we insert new bead into the sequence. Once three or more beads with a same color become adjacent due to an insertion, they will vanish immediately.

Input

  • First line gives the initial bead sequence(letter from ‘A’ to ‘Z’),
  • Second line gives the number of insertions n
  • The following n lines tell all the insertions in turn. Each contains an integer k and a capital letter.

Output

The evolutionary history of the bead sequence.
Specially, “-“ stands for an empty sequence.

Restrictions

  • 0 <= n <= 10^4, 0 <= length of the initial sequence <= 10^4
  • Time: 2s
  • Memory: 256 MB

Example

  • input

  • output

Idea

Use list. First construct a linknode and then form a list class. Use check() function to check whether there are three or more adjacent beads. While checking is true, call change() function to clean the adjacent beads.
The key algorithm is the implementation of list.

THUOJ PA1 Range

Posted on 2018-08-18 | In THUOJ

Description

S is a set of n integers in the X coordinates. For a certain interval [a, b]. Calculate the number of points that is inside the interval.

Input

  • Fisrt line: n, m (n is the size of S, m is the number of quiries)
  • Second line: enumerate all the n points in set S
  • The following m lines: each line contains integer a and b, representing interval [a, b]

Output

The number of points that is inside the interval.

Restrictions

  • 0 <= n, m <= 5*10^5, a <= b, the point inside S is unique. a and b are nonnegative integers and they are not larger than 10^7
  • Time: 2s
  • Memory: 256 MB

Idea

First consider brute-force solution. Judge each element in S whether it is inside the interval [a, b]. Time complexity is O(n). It is not ideal when the test data is too large.
So consider use binary search for the start point and end point of the interval. We get minIndex and maxIndex. The key to solve the problem is carefulness. The search for two points shouls be implemented seperately, because when the interval point is not in the set S, the indices they returned are different.
