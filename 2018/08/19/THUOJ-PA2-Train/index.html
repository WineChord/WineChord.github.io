THUOJ PA2 Train

发表于 2018-08-19 | 分类于 THUOJ

Description

Figure below shows the structure of a station for train dispatching

A is the entrance and B is the exit. S is the transfer end. All single tracks are one-way, which means that the train can enter the station from A to S, and pull out from S to B. Note that the overtaking is not allowed. Because of the limited capacity of S, no more than m compartments can reside at S simultaneously.
Assume that a train consist of n compartments labeled {1,2,…,n}. A dispatcher wants to know whether these compartments can pull out at B in the order of {a1, a2, … an}(a sequence). If can, in what order he should operate it?

Input

  • 1st line: two integers n and m.
  • 2nd line: n integers separated by spaces, which is a permutation of {1,2,…,n}. This is a compartment sequence that is to be judged regarding the feasibility.

Output

  • If feasible, output the sequence of operations using “push” and “pop”
  • If infeasible, output a “No”

Idea

Trivial thought is that enumerate all the possibilities and see whether the output sequence is possible. However time consumption will be unbearable.
In my code I use one stack to simulate the stack permutation.

  • given one number in the left list, check whether the top of the Stack equals it.
  • If not, push another number into the stack until check succeed or stack is full or numbers are exhausted.

Note that we only need to check whether the top of the stack equals to current number instead of check the whole stack.
