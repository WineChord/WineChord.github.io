THUOJ PA1 Range

Posted on 2018-08-18 | In THUOJ

Description

S is a set of n integers in the X coordinates. For a certain interval [a, b]. Calculate the number of points that is inside the interval.

Input

  • Fisrt line: n, m (n is the size of S, m is the number of quiries)
  • Second line: enumerate all the n points in set S
  • The following m lines: each line contains integer a and b, representing interval [a, b]

Output

The number of points that is inside the interval.

Restrictions

  • 0 <= n, m <= 5*10^5, a <= b, the point inside S is unique. a and b are nonnegative integers and they are not larger than 10^7
  • Time: 2s
  • Memory: 256 MB

Idea

First consider brute-force solution. Judge each element in S whether it is inside the interval [a, b]. Time complexity is O(n). It is not ideal when the test data is too large.
So consider use binary search for the start point and end point of the interval. We get minIndex and maxIndex. The key to solve the problem is carefulness. The search for two points shouls be implemented seperately, because when the interval point is not in the set S, the indices they returned are different.
Initially I thought that the set S given is sorted. However, some test data are not sorted. So we need to sort the set at the beginning.
